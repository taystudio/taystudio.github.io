— 브라우저 안에서만 처리

PDF가 외부 서버로 전송되지 않습니다 — 계약서·이력서·민감 문서 안심.

SmallPDF·iLovePDF·Adobe 온라인 도구는 클라우드 업로드 방식이지만, 본 사이트는 모든 처리가 사용자 브라우저 메모리에서만 일어납니다. PDF 합치기·자르기·편집(페이지 삭제·순서 변경·회전)·이미지↔PDF 변환·워터마크/페이지번호·비밀번호 설정/해제까지 9가지 도구를 무료·가입 없이 사용 가능합니다. 워터마크·로고·광고도 결과 파일에 자동 추가되지 않으며, 처리 라이브러리는 모두 vendoring(직접 commit)되어 외부 CDN 의존도 없습니다.

최종 갱신: 2026-05-17 도구 9선 파일 외부 전송 X

처리 방식·기술

누가 어떻게 쓰나
이력서·자소서여러 PDF 합치기로 이력서 + 자소서 + 포트폴리오 1 파일화. 사진 페이지를 PDF로 변환. 파일 크기 압축 안 되는데 페이지 삭제로 용량 축소.
계약서·법무계약서 PDF에 비밀번호 + 인쇄·복사·편집 권한 제한 (RC4 128bit). 워터마크로 "사본·draft·confidential" 표시. 페이지 번호 자동 매김.
학생·연구자여러 강의 자료 합치기, PDF에서 필요한 페이지만 추출(범위 1-3,5,7-10), PDF → PNG로 이미지 추출해 슬라이드 자료 활용.
중고거래·민감 문서주민등록증·신분증 사본 PDF에 모자이크 없이 워터마크로 "거래용·재발급 불가" 표시. PDF 메타데이터(저자·기관) 정리.
본인 소유 PDF 비번 해제오래된 본인 소유 암호 PDF의 비밀번호를 본인이 알고 있을 때 — 비밀번호 입력 후 무손실(텍스트·이미지·하이퍼링크) 재저장. 무차별 대입 X.
이미지 PDF화JPG·PNG 사진을 PDF로 변환해 인쇄. A4·Letter·원본 사이즈 옵션. 여러 페이지 PDF 생성.
8 도구의 라이브러리 구성
  • 합치기·자르기·편집·이미지→PDF — pdf-lib (MIT, JavaScript)
  • 편집 썸네일·PDF→이미지 — pdf.js (Mozilla, Apache-2.0)
  • 비밀번호 설정 — pdf-lib-plus-encrypt CDN (RC4 128bit). 인쇄·복사·편집·주석 권한 4종 제한 옵션
  • 비밀번호 제거 — qpdf (Apache 2.0, WebAssembly)로 비밀번호 검증·복호화. 암호화 레이어만 제거하고 페이지 구조는 보존 → 텍스트·이미지·하이퍼링크 모두 유지
  • 다중 파일 batch — pdf-merge·pdf-stamp·img-to-pdf 모두 합계 100MB total 가드

모든 라이브러리는 사이트에 vendoring(직접 commit)되어 외부 CDN 의존도가 없습니다. 외부 네트워크 오프라인 상태에서도 도구 동작 가능.

비밀번호 보안 — 진짜 안전한가요?

RC4 128bit는 PDF 표준 암호화 알고리즘이지만, 1990년대 후반 표준이라 현대 보안 수준에서는 약한 편입니다. 강한 보안이 필요한 문서(영업비밀·법적 증거)는 외부 zip 7z AES-256 또는 전용 DRM 솔루션을 권장합니다. 일반적인 권한 제한(인쇄·복사·편집 차단)·암호 보호 목적에는 충분합니다.

PDF 표준이 AES-256를 지원하지만, pdf-lib-plus-encrypt 라이브러리는 RC4 128bit만 구현되어 있습니다. 추후 AES 지원 라이브러리 검토 예정.

PDF 크기 한계와 모바일 대응

데스크톱 Chrome 기준 200MB·200쪽까지 무난. 모바일 Safari·Chrome은 메모리 한계로 50MB·50쪽 권장. 큰 PDF는 PDF 자르기로 범위를 좁힌 후 작업하세요. 편집 도구는 썸네일 생성으로 더 보수적인 한계 (100쪽+ confirm·200쪽+ 강화 confirm).

결과 PDF에 폰트·이미지·하이퍼링크는 보존되나요?

pdf-lib는 페이지 객체를 원본 그대로 복사하므로 폰트·이미지·하이퍼링크 대부분 유지됩니다. 단 PDF 폼 인터랙티브 동작·디지털 서명·일부 주석은 변경 후 동작이 달라질 수 있습니다. 비밀번호 제거는 qpdf가 암호화 레이어만 벗기므로 텍스트·이미지·하이퍼링크 무손실 — 이미지화로 처리하지 않습니다.

자주 묻는 질문

업로드한 PDF가 정말 외부로 안 가나요?
네. 모든 처리는 사용자 브라우저의 JavaScript로만 일어납니다. PDF 합치기·자르기·편집·이미지→PDF는 pdf-lib(MIT), PDF→이미지·편집 썸네일은 pdf.js(Mozilla, Apache-2.0)로 클라이언트 사이드 동작. 사이트 운영자도 파일 내용을 볼 수 없으며 외부 서버로 전송되지 않습니다.
암호 걸린 PDF도 작업되나요?
합치기·자르기·편집·변환 도구는 암호로 잠긴 PDF를 바로 처리하지 못합니다. 비밀번호를 알고 있다면 'PDF 비밀번호 제거' 도구로 먼저 암호를 풀고(qpdf로 복호화) 다시 업로드하세요. 비밀번호를 모르는 PDF는 풀 수 없습니다.
큰 PDF는 어디까지 처리되나요?
데스크톱 Chrome 기준 200MB·200쪽까지 무난. 모바일은 50MB·50쪽 권장. 메모리 부족 시 PDF 자르기로 범위를 좁힌 후 작업하세요. 편집 도구는 썸네일 생성 때문에 더 보수적인 한계입니다.
결과 PDF에 폰트·이미지·하이퍼링크는 보존되나요?
pdf-lib는 페이지 객체를 원본 그대로 복사하므로 폰트·이미지·하이퍼링크 대부분 유지됩니다. 단 PDF 폼 인터랙티브 동작·디지털 서명·일부 주석은 변경 후 동작이 달라질 수 있습니다.
모바일에서 동작이 불안정해요
모바일 Safari·Chrome은 메모리 한계가 ~500MB 정도라 큰 PDF는 데스크톱 권장. 또한 카톡·라인 등 in-app 브라우저는 동영상·PDF 처리에 부적합 — '외부 브라우저로 열기' 옵션을 사용하세요(자동 안내 배너 노출).
