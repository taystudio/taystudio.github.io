— 브라우저 안에서만 처리

PDF가 외부 서버로 전송되지 않습니다 — 계약서·이력서·민감 문서 안심.

SmallPDF·iLovePDF·Adobe 온라인 도구는 클라우드 업로드 방식이지만, 본 사이트는 모든 처리가 사용자 브라우저 메모리에서만 일어납니다. PDF 합치기·자르기·편집(페이지 삭제·순서 변경·회전)·이미지↔PDF 변환까지 5가지 도구를 무료·가입 없이 사용 가능합니다. 워터마크·로고·광고도 결과 파일에 추가되지 않으며, 처리 라이브러리는 모두 vendoring(직접 commit)되어 외부 CDN 의존도 없습니다.

자주 묻는 질문

업로드한 PDF가 정말 외부로 안 가나요?
네. 모든 처리는 사용자 브라우저의 JavaScript·WebAssembly로만 일어납니다. PDF 합치기·자르기·편집·이미지→PDF는 pdf-lib(MIT), PDF→이미지·편집 썸네일은 pdf.js(Mozilla, Apache-2.0)로 클라이언트 사이드 동작. 사이트 운영자도 파일 내용을 볼 수 없으며 외부 서버로 전송되지 않습니다.
암호 걸린 PDF도 작업되나요?
암호로 잠긴 PDF는 자동으로 처리되지 않을 수 있습니다. 본 사이트는 잠금 해제(decrypt)를 지원하지 않습니다. PDF 뷰어에서 암호 해제 후 다시 업로드하세요. 단순히 ignoreEncryption 플래그로 일부 가벼운 보호는 우회되지만, 강한 암호는 동작 X.
큰 PDF는 어디까지 처리되나요?
데스크톱 Chrome 기준 200MB·200쪽까지 무난. 모바일은 50MB·50쪽 권장. 메모리 부족 시 PDF 자르기로 범위를 좁힌 후 작업하세요. 편집 도구는 썸네일 생성 때문에 더 보수적인 한계입니다.
결과 PDF에 폰트·이미지·하이퍼링크는 보존되나요?
pdf-lib는 페이지 객체를 원본 그대로 복사하므로 폰트·이미지·하이퍼링크 대부분 유지됩니다. 단 PDF 폼 인터랙티브 동작·디지털 서명·일부 주석은 변경 후 동작이 달라질 수 있습니다.
모바일에서 동작이 불안정해요
모바일 Safari·Chrome은 메모리 한계가 ~500MB 정도라 큰 PDF는 데스크톱 권장. 또한 카톡·라인 등 in-app 브라우저는 동영상·PDF 처리에 부적합 — '외부 브라우저로 열기' 옵션을 사용하세요(자동 안내 배너 노출).
