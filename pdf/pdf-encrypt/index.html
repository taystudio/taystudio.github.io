RC4 128-bit 암호화 — 실제 보안 강도

PDF 1.7 표준 암호화 방식 (RC4 128-bit). Adobe Acrobat·Preview·Chrome PDF 뷰어 등 모든 표준 PDF 뷰어와 호환됩니다.

강점 — 일반적인 권한 제한(인쇄·복사·편집 차단)·암호 보호 목적에는 충분. 호환성 최고.

약점 — 1990년대 후반 표준이라 강한 보안에는 부적합. 영업비밀·법적 증거·국가기밀 등은 외부 zip 7z AES-256 또는 전용 DRM을 권장합니다.

PDF 표준은 AES-256를 지원하지만, 본 도구가 사용하는 pdf-lib-plus-encrypt 라이브러리는 RC4 128bit만 구현. AES 지원 라이브러리 검토 예정.

권한 옵션 4종 상세
  • 인쇄 권한 차단 — 비밀번호 없이 열어도 인쇄 X (일부 뷰어는 무시할 수 있음)
  • 복사 권한 차단 — 본문 텍스트 선택·복사 X. 스크린샷·OCR로 우회 가능
  • 편집 권한 차단 — 페이지 추가·삭제·수정 X
  • 주석 권한 차단 — 형광펜·메모·서명 X

스크린리더 접근성(contentAccessibility)은 항상 true로 유지됩니다. 시각장애인 보조 기술이 텍스트를 읽을 수 있도록 보장.

비밀번호 입력 안전망
  • 2회 입력 + 실시간 일치 검증 — 입력 중 일치 여부가 색깔로 표시 (초록 vs 빨강)
  • 4자 이상 — 너무 짧은 비번은 거부 (브루트포스 1초 내 가능)
  • 표시/숨김 토글 — 입력 확인용 보기 버튼
  • 이미 암호화된 PDF 감지 — 재암호화 시도하면 명시 alert
  • 비번 분실 시 복구 불가 — 본 도구·운영자 모두 비번 복구 불가. 안전한 곳에 메모 권장
흔한 실수·주의점
  • 약한 비밀번호 (1234·password) — 1초 내 brute-force 뚫림. 영문 대소문자 + 숫자 + 특수문자 12자 이상 권장.
  • 비번 분실 — 본 도구는 복구 기능 없음. 비번 메모·관리자 password 옵션 X.
  • 이메일에 비번 함께 보내기 — 같은 이메일에 PDF + 비번 보내면 의미 없음. 비번은 카톡·전화 등 다른 채널로.
  • 인쇄 차단 우회 — Chrome "PDF로 인쇄" 또는 스크린샷으로 차단 우회 가능. 권한 제한은 정직한 사용자에 한정.
  • RC4 권한 무력화 도구 — 일부 공개 도구로 권한 차단 해제 가능. 권한은 일차적 방어 수준.

자주 묻는 질문

PDF와 비밀번호가 외부로 전송되나요?
전송되지 않습니다. 모든 암호화는 브라우저 pdf-lib(JavaScript 라이브러리)에서. 비밀번호·PDF 내용 모두 사이트 운영자도 볼 수 없습니다.
어떤 암호화 방식인가요?
RC4 128-bit (PDF 1.7 표준). Adobe Acrobat·Preview·Chrome PDF 뷰어 등 모든 표준 뷰어와 호환.
비밀번호를 잊으면?
복구 불가. 비밀번호는 브라우저에서만 쓰이고 어디에도 저장되지 않습니다. 별도 저장 필수.
사용자 비밀번호와 소유자 비밀번호 차이는?
사용자 비밀번호: PDF 열 때 필요. 소유자 비밀번호: 권한 제한 해제 시 필요. 본 도구는 둘 다 같은 비밀번호로 설정.
권한 제한은 얼마나 강력한가요?
PDF 뷰어가 자발적으로 따르는 표준. Adobe Acrobat·Chrome·Preview 등 주류 뷰어는 모두 준수하지만, 일부 서드파티 도구는 우회 가능. 강한 보안이 필요하면 사용자 비밀번호도 함께 설정.
이미 비밀번호가 걸린 PDF에 또 걸 수 있나요?
불가. 기존 비밀번호를 먼저 해제해야 합니다. 본 도구는 평문 PDF만 받습니다.
본인 소유 PDF만 사용하세요
타인의 PDF에 임의로 비밀번호를 거는 행위는 저작권·업무 방해 등 법적 문제가 될 수 있습니다. 본인 소유 또는 권한이 있는 PDF만 암호화하세요.
