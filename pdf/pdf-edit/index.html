알아두면 좋은 것

  • 페이지 단위 편집입니다 — 페이지 삭제·순서 변경·회전은 되지만, 페이지 안의 글자·이미지를 직접 수정하지는 못합니다. PDF는 글자가 흐르는 텍스트가 아니라 좌표에 고정된 형태라, 내용을 바꾸려면 PDF 편집·주석으로 흰색으로 가린 뒤 새 텍스트를 얹는 방식이 필요합니다.
  • 암호가 걸린 PDF는 먼저 비밀번호 제거로 푼 뒤 편집하세요 — 암호화된 파일은 페이지 구조를 읽지 못합니다.
  • 썸네일은 미리보기용으로 저해상도 렌더입니다. 실제 결과 화질은 원본 그대로 유지되며, 저장 후 PDF 뷰어에서 페이지 순서·회전을 최종 확인하는 것을 권장합니다.
  • 원본은 보존 — 편집 결과는 새 파일로 저장되고, 업로드한 원본과 모든 작업은 브라우저 안에서만 처리됩니다.

썸네일이 흐릿한데 결과 화질도 그런가요?
썸네일은 빠른 미리보기를 위해 낮은 해상도(가로 ~150px)로 그립니다. 저장된 PDF는 원본 페이지를 그대로 복사하므로 화질·텍스트·폰트 손실이 없습니다.
회전이 PDF 파일 크기에 영향을 주나요?
거의 영향 없습니다. pdf-lib는 페이지의 회전 메타데이터(/Rotate)만 변경하므로 픽셀 데이터를 다시 인코딩하지 않습니다. 결과 파일 크기는 원본과 거의 동일.
드래그가 모바일에서 동작 안 해요
모바일 브라우저는 HTML5 드래그&드롭이 제한적입니다. 각 페이지 카드의 ↑·↓ 버튼으로 동일한 순서 변경이 가능합니다. 큰 이동은 여러 번 누르세요.
pdf-lib·pdf.js 라이브러리
pdf-lib (Hopding, MIT) = PDF 생성·수정 / pdf.js (Mozilla, Apache-2.0) = 썸네일 렌더링. 둘 다 vendoring(직접 commit)으로 외부 CDN 의존 없음.

자주 묻는 질문

PDF 파일이 외부 서버로 업로드되나요?
전송되지 않습니다. PDF 처리는 브라우저 내부의 pdf-lib·pdf.js 라이브러리에서 모두 수행되며, 사이트 운영자도 파일 내용을 볼 수 없습니다. 계약서·이력서·민감 문서도 안심하고 사용하세요.
페이지를 잘못 삭제했어요. 되돌릴 수 있나요?
네. 삭제된 페이지는 회색으로 표시되며 ↺ 버튼으로 즉시 복원됩니다. 저장 버튼을 누르기 전까지는 모든 변경(삭제·순서·회전)이 메모리에 머물러 있어 자유롭게 되돌릴 수 있습니다.
회전은 어떻게 작동하나요?
각 페이지의 ↻ 버튼을 한 번 누를 때마다 시계방향으로 90도 회전합니다. 90·180·270·0(원래) 순환. 회전은 PDF 메타데이터 수준에서 처리되어 결과 파일 크기가 거의 늘지 않습니다.
모바일에서 페이지 순서 변경은 어떻게 하나요?
각 페이지 카드의 ↑·↓ 버튼으로 순서를 바꿀 수 있습니다. 데스크톱에서는 추가로 드래그·드롭도 지원합니다. 큰 이동은 ↑·↓ 버튼이 더 정확하니 모바일에서도 그대로 사용하세요.
최대 몇 페이지까지 처리되나요?
썸네일 미리보기 때문에 페이지 수가 많을수록 처음 로딩이 길어집니다. 데스크톱 Chrome 기준 100~200페이지까지 무난, 모바일은 50페이지 이하 권장. 큰 PDF는 먼저 PDF 자르기로 범위를 좁힌 후 편집하세요.
암호 걸린 PDF도 편집되나요?
암호로 잠긴 PDF는 자동으로 처리되지 않을 수 있습니다. 본 도구는 잠금 해제를 지원하지 않습니다. PDF 뷰어에서 암호 해제 후 다시 업로드하세요.
