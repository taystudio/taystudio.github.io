합치기 전 알아두면 좋은 것

  • 올린 순서 = 합쳐지는 순서 — 목록에서 위에 있는 파일이 앞쪽 페이지가 됩니다. ↑↓ 버튼으로 순서를 맞춘 뒤 합치세요.
  • 암호가 걸린 PDF는 합쳐지지 않습니다 — 먼저 비밀번호 제거로 푼 뒤 다시 올리세요.
  • 용량은 대략 합산됩니다(10MB + 10MB ≈ 20MB). 합본이 너무 크면 먼저 자르기로 필요한 페이지만 추출하면 가볍습니다.
  • 글꼴·이미지·링크 보존 — 원본 페이지 객체를 그대로 복사하므로 글꼴·이미지·대부분의 하이퍼링크가 유지됩니다. 다만 인터랙티브 폼·디지털 서명은 합본 후 동작이 달라질 수 있습니다.
  • 페이지 수 제한 없음 — 수십 개 파일도 한 권으로 합칠 수 있습니다. 다만 총 용량이 100MB를 넘으면 기기 메모리에 따라 느려질 수 있어 확인 메시지가 표시됩니다.

pdf-lib · pdf.js 라이브러리
pdf-lib (Hopding, MIT) = 페이지 병합·복사. pdf.js (Mozilla, Apache-2.0) = 「페이지 선택 합치기」의 페이지 썸네일 미리보기 렌더링. 둘 다 vendoring(직접 commit)이라 외부 CDN 의존이 없고, 클라이언트에서 100% 동작합니다. 썸네일은 미리보기를 열 때만 파일별로 생성돼 부하를 최소화합니다.
큰 파일 처리 시 주의
PDF 처리 중에는 탭 메모리가 일시적으로 크게 증가합니다. 모바일 Safari는 ~200MB 근처에서 강제 종료될 수 있으니 큰 파일은 데스크톱에서 작업 권장. 처리 시간은 페이지 수보다 임베디드 이미지·폰트 양에 더 좌우됩니다.
파일 안에서 원하는 페이지만 골라 합칠 수 있나요?
네. 상단 「페이지 선택 합치기」 탭을 쓰면 파일마다 페이지 범위(예: 1-3,5,7-10)를 지정해 그 페이지만 뽑아 합칩니다. 비워두면 그 파일은 전체 페이지가 들어갑니다. 실제 페이지 수를 넘는 범위는 자동으로 무시됩니다. 단, 합친 후의 페이지 재배치는 지원하지 않으니, 순서가 복잡하면 필요한 부분을 각 파일에서 미리 골라 순서대로 올리세요.
워터마크가 붙나요?
붙지 않습니다. 본 도구는 라이브러리 광고나 워터마크를 추가하지 않으며, AdSense 광고는 페이지 UI 영역에만 표시됩니다. 결과 PDF는 깨끗합니다.

자주 묻는 질문

PDF 파일이 외부 서버로 업로드되나요?
전송되지 않습니다. PDF 처리는 브라우저 내부의 pdf-lib·pdf.js 라이브러리에서 모두 수행되며, 사이트 운영자도 파일 내용을 볼 수 없습니다. 계약서·이력서·민감 문서도 안심하고 사용하세요.
암호 걸린 PDF도 합쳐지나요?
암호로 잠긴 PDF는 자동으로 처리되지 않을 수 있습니다. 잠금을 먼저 해제한 후 업로드하세요. 암호 해제는 본 도구에서 지원하지 않으며, 원본 파일의 권한을 가진 사용자가 PDF 뷰어에서 직접 해제해야 합니다.
파일 순서를 바꿀 수 있나요?
네. 업로드된 파일 목록에서 ↑·↓ 버튼으로 순서를 조정할 수 있습니다. 순서대로 페이지가 이어붙어 1개의 PDF가 됩니다. 필요 없는 파일은 ✕ 버튼으로 제거.
최대 몇 개까지 합칠 수 있나요?
이론적으로 제한 없으나 브라우저 메모리에 좌우됩니다. 데스크톱 Chrome 기준 총 200MB·100페이지 정도까진 무난. 모바일은 50MB 이하·30페이지 권장. 큰 파일은 먼저 PDF 자르기로 페이지 수를 줄이거나 데스크톱에서 처리하세요.
원본 PDF의 폰트·이미지·하이퍼링크는 보존되나요?
pdf-lib는 페이지를 원본 그대로 복사하므로 폰트·이미지·하이퍼링크 대부분 유지됩니다. 단 PDF 폼 필드의 인터랙티브 동작·디지털 서명·일부 주석은 합치기 후 동작이 달라질 수 있습니다.
결과 파일이 원본보다 용량이 큰데 정상인가요?
정상 가능 범위입니다. pdf-lib는 페이지 객체를 그대로 복사하므로 원본의 폰트·이미지가 중복 저장될 수 있습니다. 용량이 너무 크면 합친 후 다른 PDF 압축 도구를 추가로 사용하거나, 원본 단계에서 이미지 해상도를 낮추는 방법이 있습니다.
