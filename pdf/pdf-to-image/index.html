PNG vs JPG 정확히 어떤 차이?
PNG는 무손실 압축이라 텍스트·로고·도형 가장자리가 또렷합니다. JPG는 손실 압축이라 사진은 좋지만 글자 가장자리에 색번짐(artifact)이 생길 수 있습니다. 스캔된 사진 페이지 = JPG, 문서·도형 페이지 = PNG가 일반 가이드.
왜 변환에 시간이 걸리나요?
pdf.js가 각 페이지를 브라우저 canvas에 그린 다음 PNG/JPG로 인코딩합니다. 페이지 수·DPI에 비례해 시간 증가. 600 DPI는 150 DPI 대비 약 16배 픽셀이라 그만큼 오래 걸립니다. 모바일은 50페이지·300 DPI 이하 권장.
변환된 이미지가 흐려요
DPI를 높이세요. 기본 150 DPI는 화면용입니다. 인쇄·확대용은 300~600 DPI 권장. 단 원본 PDF 자체가 저해상도 스캔본이라면 DPI 올려도 더 선명해지지 않습니다.
pdf.js 라이브러리
pdf.js (Mozilla, Apache-2.0). Firefox 내장 PDF 뷰어로 쓰이는 정식 라이브러리. 본 사이트는 vendoring(직접 commit)으로 외부 CDN 의존 없음.

자주 묻는 질문

PDF 파일이 외부 서버로 업로드되나요?
전송되지 않습니다. PDF 변환은 브라우저 내부의 pdf.js 라이브러리에서 모두 수행되며, 사이트 운영자도 파일 내용을 볼 수 없습니다. 계약서·이력서·민감 자료도 안심하고 사용하세요.
PNG와 JPG 중 어느 것을 골라야 하나요?
텍스트·도형·로고가 많은 페이지는 PNG(무손실, 선명) 권장. 사진·스캔본은 JPG(용량 5~10배 작음) 권장. SNS 업로드·이메일 첨부에는 JPG가 일반적으로 더 효율적입니다.
DPI는 얼마로 설정해야 하나요?
화면 보기 = 72~150 DPI, 일반 인쇄 = 150~300 DPI, 고품질 인쇄 = 300~600 DPI. 기본값 150 DPI는 화면·SNS·간단한 인쇄까지 무난. DPI를 올리면 이미지 가로·세로가 커지고 용량이 제곱으로 증가하니 필요한 만큼만.
페이지 범위는 어떻게 입력하나요?
1쪽부터 시작. 단일 페이지 = 숫자(예: 5), 연속 = 하이픈(예: 3-7), 여러 묶음 = 콤마(예: 1-3,5,8-10). 비워두면 전체 페이지 변환. 공백은 자동 무시.
한 번에 모든 페이지를 다운로드할 수 있나요?
변환 후 페이지별 다운로드 링크가 표시됩니다. '📦 ZIP으로 묶어 다운로드' 버튼이 권장 — 파일 1개로 모든 페이지가 묶여 받아집니다. 개별 다운로드를 원하면 '전체 다운로드 (개별)'을 누르세요. 브라우저가 5개 이상 동시 다운로드를 차단할 경우 권한을 허용하거나 페이지별로 클릭하세요.
이미지에서 텍스트를 추출하려면?
본 도구는 페이지 시각 자체를 이미지로 변환합니다. 이미지에서 텍스트(OCR)가 필요하면 사이트의 '이미지 글자 추출(OCR)' 도구를 추가로 사용하세요. 단 PDF가 텍스트 기반이면 PDF에서 직접 복사·붙여넣기가 더 정확합니다.
