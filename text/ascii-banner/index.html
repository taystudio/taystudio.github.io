아스키 아트(figlet)란

아스키 아트 배너는 한 글자를 여러 줄의 기호로 그려 콘솔에 뜨는 큰 글씨로 만드는 표현입니다. 1991년 Unix용 figlet(Frank, Ian, Glenn's Letters)에서 대중화됐고, Standard·Slant·Big·Shadow 같은 글꼴이 여기서 나왔습니다. 이 도구는 figlet 폰트 8종을 브라우저에서 렌더링합니다.

폰트느낌추천 용도
Standard기본 균형형범용 README 제목
Big / Banner3크고 두꺼움강조 로고, 채울 문자와 조합
Slant기울임속도감 있는 로고
ANSI Shadow입체 그림자CLI 스플래시 배너
Doom / Ghost굵은 장식체레트로·게임 느낌
Small작고 컴팩트좁은 폭·코드 주석

⚠️ figlet 폰트는 영문·숫자·기호만 그립니다. 한글·한자는 변환되지 않고 그대로 출력됩니다(영문 로고용).

왜 폰트마다 너비·높이가 다른가 — FIGfont 구조

같은 글자라도 폰트에 따라 크기·정렬이 다른 데는 이유가 있습니다. FIGfont(.flf)는 글자 하나를 고정 높이의 여러 줄(sub-character rows) 격자로 정의해요. 이 도구에 든 폰트의 실제 높이를 보면 — Standard·Slant 6줄, ANSI Shadow 7줄, Big·Doom 8줄, Small 5줄. 그래서 폰트를 바꾸면 전체 높이가 통째로 달라집니다.

가로 폭은 레이아웃 규칙이 정합니다. figlet 기본값은 smushing(인접 글자의 경계 칸을 규칙대로 겹쳐 압축)이라 글자들이 딱 붙어 좁아져요. 폰트마다 smushing 규칙(hardblank·겹침 허용 비트)이 달라서, 같은 단어도 ANSI Shadow는 넓고 Small은 좁게 나옵니다.

  • 고정폭(monospace)이 필수인 이유 — 각 글자가 N줄 격자라 줄마다 칸 수가 정확히 맞아야 위아래가 정렬됩니다. 가변폭 글꼴은 칸 폭이 글자마다 달라 격자가 어긋나 깨집니다.
  • 긴 텍스트가 잘리는 이유 — figlet은 자동 줄바꿈을 안 하므로 (글자 수 × 글자폭)이 화면을 넘으면 가로로 길어지거나 잘립니다. 짧은 이름·이니셜이 안전합니다.
  • 이모지 채움이 밀리는 이유 — 이모지는 한 글자가 보통 2칸 폭이라, smushing으로 압축된 격자에서 칸이 밀려 정렬이 흐트러질 수 있습니다(굵은 폰트일수록 여유 있음).

이 도구는 figlet 기본 textSync(smushing) 동작을 그대로 씁니다 — 터미널 figlet 명령과 같은 결과가 나옵니다.

언제 쓰나
오픈소스 README프로젝트 이름을 큰 ASCII 로고로 만들어 README 최상단에 배치. 코드 블록으로 감싸 정렬 유지.
CLI 시작 배너커맨드라인 도구 실행 시 뜨는 splash 배너. ANSI Shadow·Big이 인기.
코드 주석 구분긴 소스의 섹션을 큰 글씨 주석으로 구분해 가독성 향상.
디스코드·슬랙코드 블록(```)으로 감싸 큰 글씨 메시지·서버 배너로 활용.
이모지 블록 로고채울 문자에 🔥·★·█를 넣어 블록·이모지 느낌의 눈에 띄는 로고 제작.
레트로 콘솔 꾸미기Doom·Ghost 같은 장식 폰트로 복고풍 터미널 분위기 연출.
채울 문자 — 획을 다른 기호로

'채울 문자'에 한 글자를 넣으면, figlet이 그린 글자 획을 모두 그 문자로 치환합니다. 비우면 폰트 원본 그대로입니다. 예를 들어 Banner3 폰트에 🔥를 채우면 불꽃 블록 로고가 됩니다. 가는 폰트(Standard·Small)보다 굵은 블록 폰트(ANSI Shadow·Banner3·Big)에서 효과가 큽니다.

figlet 라이브러리·폰트 self-host

이 도구는 figlet.js와 FIGfont 폰트 파일(.flf) 8개를 사이트에 직접 보관(self-host)합니다. 외부 CDN을 거치지 않아 변환이 브라우저 안에서만 일어나고, CDN 장애·차단의 영향을 받지 않습니다. 폰트는 페이지에서 한 번 불러온 뒤 재사용됩니다.

흔한 실수·주의점
  • 한글 입력 — figlet 폰트엔 한글 글자 모양이 없어 변환되지 않습니다. 영문·숫자만.
  • 가변폭에서 깨짐 — 일반 글꼴로 보면 칸이 어긋납니다. 코드 블록·고정폭에서 봐야 정렬 유지.
  • 너무 긴 텍스트 — 글자가 많으면 가로로 길어져 줄바꿈·잘림이 생깁니다. 짧은 이름·이니셜 권장.
  • 이모지 채움 + 좁은 칸 — 이모지는 폭이 넓어 일부 폰트에서 칸이 밀릴 수 있습니다. 굵은 폰트에서 확인.

자주 묻는 질문

한글도 아스키 아트 배너로 만들 수 있나요?
안 됩니다. figlet 폰트(FIGfont)는 ASCII 영문 대소문자·숫자·기호에만 글자 모양이 정의되어 있어, 한글·한자·일본어를 넣으면 변환되지 않고 그대로 출력됩니다. 영문 로고·이니셜·프로젝트 이름에 사용하세요.
어디에 쓰는 도구인가요?
GitHub README 상단 제목, 터미널·CLI 프로그램 시작 배너, 코드 주석 구분선, 디스코드·슬랙 메시지, 레트로 콘솔 꾸미기 등에 씁니다. 결과를 복사해 README나 코드 블록에 붙여넣으면 큰 글씨 로고가 됩니다.
'채울 문자'는 무엇인가요?
ASCII 글자를 이루는 획 문자를 다른 문자로 바꾸는 옵션입니다. 비워 두면 폰트 원본 그대로, █·★·🔥 등을 넣으면 글자 획이 그 문자로 채워져 블록·이모지 느낌의 배너가 됩니다. 굵은 블록 폰트(ANSI Shadow·Banner3)에서 효과가 큽니다.
글자가 깨지거나 정렬이 안 맞아요.
ASCII art는 고정폭(monospace) 글꼴에서만 정렬이 유지됩니다. 가변폭 글꼴로 보면 칸이 어긋나 깨져 보입니다. README는 코드 블록(``` ```)으로 감싸고, 붙여넣는 편집기·터미널의 글꼴을 D2Coding·Consolas 같은 고정폭으로 두면 정확히 정렬됩니다.
폰트는 몇 종류이고 외부 서버를 쓰나요?
Standard·Big·Slant·ANSI Shadow·Doom·Ghost·Small·Banner3 8종을 제공합니다. figlet 라이브러리와 폰트 파일을 모두 사이트에 self-host 하므로, 변환은 외부 서버 호출 없이 브라우저 안에서만 처리됩니다.
입력한 텍스트가 어디로 가나요?
전송되지 않습니다. 텍스트→ASCII 변환은 브라우저에서 실행되며 입력 내용은 서버로 보내지 않습니다.
