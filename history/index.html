🧩 핵심 메커니즘 5가지

  • Push 모델: 변경 즉시 검색엔진 통보 → 며칠 → 몇 시간으로 단축
  • Key 파일 = 소유권 증명: https://host/{key}.txt body == key 검증. 공개돼도 안전
  • Hub & Spoke: api.indexnow.org 한 곳에 ping → Bing이 internal로 다른 엔진에 relay
  • 응답 코드: 200/202 성공, 403 keyLocation 검증 실패, 422 host 불일치, 429 rate limit
  • 큐잉 ≠ 인덱스 보장: 가속 채널일 뿐, 콘텐츠 품질 검증은 검색엔진 재량

ROOT_PAGES=("privacy" "terms")

for slug in "${ROOT_PAGES[@]}"; do
  file="${slug}/index.html"
  [ -f "$file" ] || continue
  is_noindex "$file" && continue
  URL=$(canonical_of "$file" "${DOMAIN}/${slug}/")
  emit_url "$URL" "$(last_mod "$file")"
done

🗜️ 1. video/compress/ — 동영상 압축

  • 옵션: 해상도(원본/1080/720/480/360p) + CRF 슬라이더(18~35, 기본 28) + 오디오(AAC/무손실/제거)

  • 출력 MP4(H.264 + AAC) — 카톡·이메일·SNS 호환. 진행률 + 압축 후 미리보기 + 원본/압축 용량/감소율/처리 시간

✂️ 2. video/trim/ — 동영상 자르기 (빠른 + 정확 모드)

  • 빠른 모드: -ss <s> -to <e> -i input -c copy — seek가 -i 앞 = 키프레임 단위, 1GB도 수 초
  • 정확 모드: -i input -ss <s> -to <e> -c:v libx264 -crf 23 — 디코드 후 프레임 단위 재인코딩
  • 시각 입력 파서 = HH:MM:SS / MM:SS / 초(.frac) 3가지 형식. 영상 업로드 시 미리보기 + duration 자동 표시 + "현재 재생 위치로" 버튼

🔄 3. video/rotate/ — 동영상 회전

📦 @imgly vendoring 결정 (사이트 첫 ESM)

패키지가 ESM-only(CJS·UMD 미제공)라 일반 <script> 태그로 불러올 수 없음. 결정한 패턴:

  • esm.sh의 self-contained bundle(?bundle 플래그)로 dependency까지 1 파일로 합친 후 vendoring
  • image/vendor/imgly-bg-remove.mjs 82KB (ESM, exports: removeBackground · preload · segmentForeground 등)
  • HTML에 <script type="module" src="./bg-remove.js"> 사용 — 사이트 첫 ESM module 로딩. 도구 JS 안에서 import { removeBackground } from '/image/vendor/imgly-bg-remove.mjs'
  • 모델·ONNX Runtime WASM(~43MB+)은 staticimgly.com CDN에서 첫 사용 시 다운로드 → 브라우저 IndexedDB·HTTP 캐시
  • Tesseract.js(OCR)와 동일 패턴 — 사용자 이미지는 외부로 안 나감, 모델만 inbound

기존 도구는 모두 UMD/IIFE를 defer script로 로딩했는데, ESM-only 패키지가 늘어나는 추세라 이 패턴이 향후 표준이 될 가능성. 다만 ESM은 정적 분석이 안 돼 SW 캐시·preload 처리에 추가 작업 필요(현재는 deferred 처리).

⚠️ 사용자 안내 정책
