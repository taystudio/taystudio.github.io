1. WHY 패턴 결정 — 왜 "도구 카드 + 위젯 슬롯"인가

쿠팡 파트너스에 카드를 박는 방법은 여러 갈래. 그 중 우리 도구 컨텍스트에 가장 잘 맞는 형식을 선택한 배경.

옵션이미지 출처디자인 통제전환률정책 안전채택
A. 쿠팡 자동 위젯 통째로 쿠팡 자동 X (위젯 디자인 그대로) ★ (도구 무관 광고 섞임) ✅ X
B. 쿠팡 이미지 직접 다운로드 쿠팡 임의 사용 O ★★★ ❌ 활동 정지 위험 X
C. Unsplash + 쿠팡 검색 링크 무료 라이선스 O ★★ (실상품 매칭 약함) ✅ X (1차 검토)
D. 도구 카테고리 카드 + 쿠팡 단일 상품 위젯 (240×240) 쿠팡 위젯 자동 △ (외곽만 통제) ★★★ (도구 컨텍스트 + 실상품) ✅ ✅ 채택
💡 채택 근거
  • 도구 컨텍스트 ↑ — 모니터 PPI 계산하던 사용자에게 "당신 입력값에 적합한 카테고리는 게이밍 모니터" + 실상품이 같이 보임 → 클릭률 ↑
  • 저작권 안전 — 이미지·가격은 쿠팡 위젯이 자동 표시 (운영자가 임의로 다운로드 안 함)
  • 가격 자동 갱신 — 위젯이 iframe이라 쿠팡 측에서 가격·이미지 자동 업데이트 → 유지보수 0
  • 외곽 디자인 통제 — 카테고리 헤더(이름·적합도 배지)는 운영자 디자인 → 도구 컨텍스트 메시지 그대로

⚠ 카테고리별로 바꿀 자리 (빨간 음영)
  • data-purpose = JS 매칭 키 (gaming/office/creative/secondary)
  • 이모지 · 카테고리 이름 · spec 텍스트
  • 마커 주석 COUPANG_WIDGET_* = grep으로 즉시 위치 찾는 용도

  • href = 위젯 받은 후 5단계에서 교체할 자리

JS 동작 (monitor.js)

JS는 위젯과 무관 — 입력값(인치·해상도) 기반으로 4 카드 중 가장 적합한 카드에 .match 클래스 추가 + .purpose-fit 텍스트 갱신만 함.

const card = document.querySelector(`.purpose-card[data-purpose="${key}"]`);
if (적합) card.classList.add('match');   // → "최적" 배지 + 파란 테두리 자동 노출
else      card.classList.remove('match');
card.querySelector('.purpose-fit').textContent = 적합도_텍스트;

5. LINK 추천 링크 URL 별도 받기 (CTA 버튼용)

카드 하단 "더 보기 →" 버튼은 위젯과 별개. 카테고리 검색 페이지로 보내는 추천 링크가 따로 필요.

왜 따로 받나

  • 위젯 = 단일 상품 (그 상품 페이지로 이동)
  • "더 보기" 버튼 = 그 카테고리 전체 검색 결과로 이동 → 사용자가 다른 옵션도 비교 가능 → 24시간 쿠키 안에 결제할 확률 ↑

받는 방법

  1. 쿠팡 메인에서 카테고리 키워드 검색 (예: "게이밍 모니터")
  2. 검색 결과 페이지 URL 복사 (예: https://www.coupang.com/np/search?q=게이밍+모니터&...)
  3. 쿠팡 파트너스 콘솔 → "링크 생성" → URL 붙여넣기 → 추천 링크 받기
  4. 받은 https://link.coupang.com/a/XXXXX 형식의 URL을 href에 박음

적용 — 4개 placeholder URL 교체

grep -n '#coupang-' tools/monitor/index.html
# 결과 (4줄):
# href="#coupang-gaming-link"
# href="#coupang-office-link"
# href="#coupang-creative-link"
# href="#coupang-secondary-link"

각 placeholder를 받은 추천 링크로 교체:

<a class="purpose-link"
   href="#coupang-gaming-linkhttps://link.coupang.com/a/XXXXX"
   target="_blank" rel="sponsored noopener">
  게이밍 모니터 더 보기 →
</a>
💡 rel="sponsored noopener" 필수
  • sponsored = Google이 권장하는 어필리에이트 링크 표시 (SEO 정책)
  • noopener = 새 탭 보안 (window.opener 누수 차단)
  • 이미 마크업에 박혀 있음 — 교체 시 유지

8. POLICY 정책 주의사항 — 절대 하지 말 것

쿠팡 파트너스 활동 정지 사유 = 매출 몰수. 아래 항목 위반 X.

❌ 쿠팡 상품 이미지 직접 다운로드 → 사이트 호스팅

❌ disclosure 박스 누락

추천 링크 박힌 페이지에 "쿠팡 파트너스 활동의 일환..." 문구 미표시 = 정책 위반. 모든 카드 페이지 하단에 .partner-disclosure div 필수.

❌ 가격·재고 정보 임의 표시

"최저가" "할인 중" 같은 텍스트를 운영자가 직접 적으면 X. 가격은 쿠팡 위젯이 자동 표시하는 것만 허용 (쿠팡 측에서 갱신).

❌ 클릭 유도 자극적 문구

"무조건 사야 할" "안 사면 후회" 같은 과장 X. "PPI 기준 적합 카테고리" 같이 도구 컨텍스트 정보성 문구만.

⚠ rel="sponsored" + target="_blank" + noopener 일관 적용

Google SEO 정책. 어필리에이트 링크는 rel="sponsored"로 표시 (또는 nofollow). 새 탭 열기 시 noopener 보안. 마크업에 이미 박혀 있음 — 교체 시 유지.

💡 운영자 직접 클릭·구매 금지 (자기-거래)

자기 추천 링크 본인이 클릭·구매하면 활동 정지. 디자인 검증 시엔 위젯 표시만 확인하고 클릭 X. 클릭 검증은 가족·친구 또는 별도 디바이스 IP에서.
