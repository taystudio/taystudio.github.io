2. 🟡 Medium 12건 — 무엇이 문제였고 어떻게 fix했나

#도구문제fix
M1format-convert 진행률 file count 기반 — 1MB×4 + 100MB×1 같은 케이스에서 잘못된 ETA. 큰 파일 시 "멈췄나?" 의심 totalBytes reduce + processedBytes 누적 → "3/5 — 5.2MB / 18.0MB" byte 비율
M2format-convert 같은 이름 5개 입력 → 결과 다운로드 명 모두 동일 → 폴더 overwrite nameMap으로 중복 카운트 → 둘째부터 -1, -2 자동 suffix
M3format-convert GIF 입력 시 canvas drawImage가 첫 프레임만 — silent data loss .gif 감지 → 첫 1회 alert "GIF는 첫 프레임만 변환됩니다 (동영상 도구 권장)". gifWarned 플래그
M4compress PNG 선택 시 quality slider opacity 0.4 + disabled — 왜 비활성인지 모름 qualityHint 동적 텍스트: "PNG는 무손실 포맷이라 품질 슬라이더가 적용되지 않습니다"
M6watermark text ↔ image 모드 전환 시 사용자가 현재 모드 잊음 (입력 사라진 듯한 혼란) 활성 panel에 primary-color 박스 "✓ 현재 [텍스트/이미지(로고)] 모드 사용 중" hint
M7resize · crop input max 없음 — 50000 입력 가능 → canvas 16384 한계 overflow silent HTML max="16000" + JS MAX_DIM 검증 + 초과 alert
M8crop 16×16px 미만 영역 선택 → silent 처리 → 사용자는 왜 안 되는지 모름 applyCrop 시 MIN_SIZE 미만 alert "최소 16×16 픽셀 이상으로..."
M9id-photo whitebg toggle 반복 시 ctx.fillRect 누적 가능성 (defensive) output ctx 시작에 ctx.clearRect(0,0,w,h) 명시
M11bg-remove .mp4를 .png로 rename + file.type 위조 → ONNX 처리 시도 → fail file.type + 파일명 확장자 둘 다 검증. MIME 스푸핑 차단
M13pdf-edit 1000+ 페이지 thumbnail 동시 render → 브라우저 OOM crash numPages > 100 confirm. > 200 강화 경고 + 분할 도구 안내
M14pdf-to-image 50+ 결과 다운로드 a.click() 순차 → 10초+ 지연 + 브라우저 차단 가능 resultUrls.length ≥ 30 시 사용자 confirm "N초+ 소요. 계속?"
M15video/to-gif 60s × 30fps × 720p → 거대 메모리, 사전 알림 X → 처리 중 OOM frames × outW × outH × 4 추정. 250MB → confirm / 600MB → 강화 경고
📌 Medium 제외 (이전 phase에서 해결)
M5 watermark JPG only → P2-4에서 PNG/WebP 옵션 추가 완료 · M10 ocr kor+eng → 이미 옵션 존재 · M12 bg-remove cancel → ONNX Runtime Web 추론 중단 불가, 영구 skip

3. ♿ a11y 강화 — 무엇이 문제였고 어떻게 fix했나

audit-2에서 a11y 카테고리 통과율 20% — 25 도구 대부분 누락. WCAG AA 미충족.

#문제fix
A1 <label class="file-drop-zone">이 키보드 사용자에게 그냥 텍스트 — Tab focus·Enter 활성 불가 모든 dropZone에 role="button" + tabindex="0" + aria-label="<도구별 안내>"
A2 dropZone Enter/Space 키 눌러도 파일 선택 다이얼로그 안 열림 JS keydown 핸들러: Enter·Space → fileInput.click(). drop·dragover 옆에 배치
A3 #result 보이기 변화가 스크린리더에 알려지지 않음 aria-live="polite" + aria-atomic="true" → 결과 표시 즉시 자동 announce
A4 진행률 bar가 시각 사용자에게만 의미 — width:50%만 가져서 스크린리더 무용 role="progressbar" + aria-valuemin/max/now + aria-label. JS setProgress에 aria-valuenow 동적 갱신
A5 progressText·metaName 등 동적 변경 텍스트가 announce되지 않음 동적 텍스트 영역에 aria-live="polite" 추가 (해당 도구)
