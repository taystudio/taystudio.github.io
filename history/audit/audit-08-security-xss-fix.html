재현 시나리오
  1. macOS/Linux에서 파일명을 <img src=x onerror="alert('xss')">.png로 변경
  2. format-convert·watermark·merge·pdf-stamp에 업로드
  3. div.innerHTML = \`...${f.file.name}...\` 시점에 HTML로 해석 → onerror 실행
왜 발생했나: sanitizeFilename은 다운로드 명만 안전화 — 화면 표시에는 적용 X. 파일 시스템 제약(<·> 차단)이 일부 OS에서만. macOS·Linux는 허용.

fix 4단계

단계변경효과
1 common/site-chrome.js에 TayStudio.escapeHtml(s) utility 추가 (5 entity: & < > " ') 공용 XSS escape API
2 8 파일에 const esc = ... 선언 + 모든 사용자 입력 변수 ${esc(...)} 처리. 14 위치 일괄 실제 XSS 차단 적용
3 fallback도 자체 escape 수행 (utility 없을 때도 5 entity replace) SW v18 옛 캐시 활성 중에도 보호
4 sw.js CACHE_VERSION v18 → v19 강제 무효화 다음 방문에 새 site-chrome.js 자동 받음

fix 코드 예 (format-convert)

// 1. utility (common/site-chrome.js)
window.TayStudio.escapeHtml = (s) => String(s == null ? '' : s)
  .replace(/&/g, '&amp;').replace(/</g, '&lt;')
  .replace(/>/g, '&gt;').replace(/"/g, '&quot;')
  .replace(/'/g, '&#39;');

// 2. 각 도구 (format-convert.js)
const esc = (window.TayStudio && window.TayStudio.escapeHtml)
  ? window.TayStudio.escapeHtml
  : (s) => String(s == null ? '' : s)
      .replace(/&/g, '&amp;').replace(/</g, '&lt;')
      .replace(/>/g, '&gt;').replace(/"/g, '&quot;')
      .replace(/'/g, '&#39;');

// 변경 전
div.innerHTML = `<div>${f.file.name}</div>`;
// 변경 후
div.innerHTML = `<div>${esc(f.file.name)}</div>`;
