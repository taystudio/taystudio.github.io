8. ✅ 검증 통과 강점

  • file size 사전 체크 utility — KO 23 + EN 23 (46 도구) · 도구별 한계 적절
  • multi-file 부분 거절 동작 (format-convert 테스트로 직접 확인 — big.png 거절 + small.png 통과)
  • OCR Tesseract.createWorker + terminate cancel UI (KO + EN)
  • video 5 도구 ffmpeg.terminate cancel UI (KO + EN = 10)
  • watermark PNG/WebP/JPG 3 출력 옵션 (KO + EN) · JPG 흰 배경 fill (투명→검은 방지)
  • i18n (한국어/영문) checkFileSize 메시지 — document.lang 또는 /en/ path 감지
  • willReadFrequently mosaic + qr-scan (실 효과 발생 영역)
  • sanitizeFilename — 다운로드 파일명 OS 호환 (Windows 예약어 처리)
  • compress 0-byte 거절 + 200MB 한계 (Playwright 실 테스트로 검증)
  • blob URL 정리 — 새 convert 시 자동 revoke
  • Playwright console error 0 (우리 코드) · 외부 AdSense·Cloudflare 한정
  • 모든 변경 JS 파일 node --check syntax pass (회귀 0)
  • IIFE 격리로 namespace 충돌 없음 (checkFileSize·sanitizeFilename·dns-prefetch 공존)
  • fallback 가드 패턴 일관 — utility 없어도 안전 (Service Worker 옛 버전 캐시 시)
