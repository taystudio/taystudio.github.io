변형 2 · 옛 도메인 https://taystudio.github.io/foo (CNAME mismatch)

DNS는 같은 GitHub Pages IP를 받지만, GitHub Pages가 Host 헤더를 보고 분기:

Browser → GitHub Pages: GET /foo · Host: taystudio.github.io
GitHub Pages → repo CNAME 파일: "taystudios.com"  ← 옛 호스트와 불일치
GitHub Pages → Browser: HTTP/2 301 Moved Permanently
                        Location: https://taystudios.com/foo
Browser ← (다이어그램 step 1로 재시작 with 신규 도메인)

근거 = repo CNAME 파일이 "taystudios.com"인 동안 GitHub Pages가 자동 처리. 6개월~1년 동안 옛 도메인 DNS 살아 있는 한 자동 redirect

💡 주요 포인트
  • Cloudflare는 DNS 해석만 — orange cloud(proxy) OFF 상태이므로 실제 트래픽은 GitHub Pages로 직접. CDN/프록시 X. (proxy ON으로 바꾸면 step 6~11이 모두 Cloudflare 엣지를 경유)
  • repo CNAME 파일 = `/CNAME` (1줄짜리, body = "taystudios.com"). GitHub Pages가 Host 헤더와 매칭하는 키. 이 파일이 도메인 단일 source
  • 인증서 = GitHub Pages가 Let's Encrypt에서 자동 발급·갱신. 별도 설정 X. Custom domain 입력 후 ~10분 내 발급
  • 실제 IP 4개(185.199.108-111.153) = GitHub Pages 공용 anycast. 가장 가까운 엣지로 자동 라우팅. dig taystudios.com / nslookup으로 직접 확인 가능
