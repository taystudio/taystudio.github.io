💡 head meta 추가 작업이 0건인 이유
msvalidate.01 코드는 Bing 계정 단위로 발급된 단일 토큰. 옛 사이트 등록 시점에 head에 박힌 코드가 신규 사이트에도 동일 매칭됨. 추가로 Import from GSC 경로는 meta 검증 자체를 우회 (GSC API 핸드셰이크). 둘 중 어느 경로든 head 변경 X.

4
URL Submission — 우선순위 10건 수동
사용자 액션 (선택) 좌측 메뉴 URL Submission

Bing 일일 한도는 10,000건이지만 UI에서 batch paste가 정상 작동하지 않는 케이스 발견 — 67건 일괄 입력 안 됨. 두 옵션 중 선택:

  • 핵심 10건만 수동 한 땀 한 땀 (메인 + 5 hub + 매출 큰 도구 4) → 5분
  • 또는 전체 skip — sitemap + IndexNow ping이 동일 효과 (다음 step 6)
⚠ 발견 — UI 일괄 입력 동작 변수
textarea가 표시되긴 하나 67줄 paste 후 Submit 시 reject 또는 silent fail 사례 있음. 매출 기여 큰 페이지 5~10건만 수동 보장하고 나머지는 IndexNow에 위임하는 게 ROI 최고.

6
IndexNow ping — 67 URL 일괄 push
자동 처리 bash scripts/indexnow-ping.sh

Bing은 IndexNow protocol 제정사 중 하나라 네이티브 우선 처리. 단일 POST로 Bing·Yandex·Naver(2024 합류)·Seznam 4 엔진 동시 push. URL Submission UI의 한도/UI 한계를 우회.

  URL count:   67

── response ──
HTTP 200
✓ ping 성공 (67 URLs queued)
✓ 결과 — HTTP 200, 67 URLs queued
분 단위로 Bing·Yandex·Naver·Seznam 봇이 신규 URL fetch 시작. URL Submission UI 일괄 안 되는 한계를 단일 명령으로 우회.
💡 commit/push 불필요
IndexNow ping은 외부 API HTTP POST일 뿐. repo 파일 수정 0건 — sitemap·robots·key 파일 다 그대로. key 파일은 이미 push 완료된 상태(2ea65ad CNAME 시점)라 검증엔진이 fetch 가능.

Bing은 자동 감지·sitemap·IndexNow가 합쳐져 있어 명시적 후속 작업 거의 없음. 모니터링 위주.

  • 1~2주 후 — Bing WMT 신규 사이트 dashboard에서 인덱싱 진행 확인 (Indexed Pages 카운트 추이)

  • 1~2주 후 — site:taystudio.github.io Bing 검색 결과 수 점진 감소 확인 (자동 이전 시그널)
  • Site Moves UI — 현 Bing UI에서 제거됨, 자동 감지로 대체. skip 확정
  • 6개월 후 — 옛 사이트(taystudio.github.io) Bing WMT 등록은 그대로 유지 가능 (자동 감지 유지용) 또는 노출 0 수렴 시 삭제

결정 의사결정 메모

Import from GSC 경로 채택

수동 등록(URL prefix + meta 검증) vs Import from GSC 두 옵션 중 후자 채택. 이유 = ① 옛 사이트가 이미 등록되어 있어 head meta 검증 실패할 일 없음 ② OAuth 동의 1번이 가장 빠름 ③ Import는 sitemap/통계 데이터까지 동기화. 수동 경로면 sitemap 따로 입력해야 함.

URL Submission 67건 batch 포기 → IndexNow로 위임

Bing UI에서 67건 paste·submit 시도했으나 silent fail. 한 땀 한 땀 67건은 시간 낭비라 판단 — 핵심 10건 수동 + IndexNow 67건 일괄로 분리. IndexNow는 Bing 네이티브이므로 효과 동일.

Site Moves UI deprecation 발견

plan.md §9.5.5에서 Bing 작업 항목으로 "Site Moves(옛→새)"가 적혀 있었으나 현 Bing UI에서 제거. 좌측 사이드바·Settings·직접 URL 모두 비표시. 자동 감지로 대체된 것으로 추정. 이번 기록으로 이후 운영자가 Site Moves 찾는 시간 낭비 방지.
