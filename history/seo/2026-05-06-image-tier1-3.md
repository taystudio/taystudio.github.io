## 신규 도구 1 — `image/heic-to-jpg/`

**vendor**: heic2any v0.0.4 (~1.35MB UMD, libheif WebAssembly 자체 번들).

**vendoring 함정 — esm.sh `?bundle` 실패**:
첫 시도 = `https://esm.sh/libheif-js@1.18.2?bundle` → 303 bytes wrapper만 받음 (`/node/buffer.mjs` 등 외부 import 그대로 둠). esm.sh는 일부 패키지에서 ?bundle이 의존성을 안 묶음. **해결** = jsdelivr `cdn.jsdelivr.net/npm/heic2any@0.0.4/dist/heic2any.min.js` 직접 다운로드 — heic2any가 자체적으로 libheif WASM을 번들해뒀음. UMD 글로벌 `window.heic2any` 사용.

**핵심 API**:
```js
const blob = await window.heic2any({
  blob: file,
  toType: 'image/jpeg',
  quality: 0.9,
});
// 다중 페이지 HEIC는 Blob[] 반환 — 첫 항목만 사용
```

**UX**:
- 다중 파일 누적 (드래그·드롭 또는 클릭, `accept=".heic,.heif,image/heic,image/heif"`)
- 포맷 = JPG (default) / PNG, 품질 = 1~100 슬라이더 (PNG 시 숨김)
- 변환 = 한 장씩 순차 처리, 진행률 + 파일 카드별 상태(대기·완료·실패) 표시
- 결과 그리드 = 각 파일 썸네일(브라우저 native rendering) + 다운로드 + "전체 다운로드" 200ms 간격 순차 anchor click
- 실패 처리 = 파일 카드에 "실패" + 에러 title, 일부 실패해도 성공한 것은 결과로 노출

**SEO 키워드 매핑**:

- description: "아이폰·아이패드 HEIC 사진을 JPG/PNG로 무료 변환. 여러 파일 일괄 처리…"
- keywords: "HEIC JPG 변환, 아이폰 사진 변환, HEIC 변환, 아이패드 사진 JPG, HEIC PNG…"
- 사용 사례 표 = 카톡/이메일·블로그·인쇄·윈도우 PC·편집·웹디자인 6행

## 신규 도구 2 — `image/crop/`

**vendor**: 없음 (canvas 2D API + Pointer Events 직접 구현)

**상태 모델**:
```js
let originalImg, transformedCanvas;
let rotation = 0;       // 0/90/180/270
let flipH = false, flipV = false;
let crop = { x, y, w, h };  // transformedCanvas 픽셀 기준
let aspectRatio = 0;    // 0 = 자유, > 0 = 비율 강제
```

**`transformedCanvas` 패턴**: 회전·반전 적용 시 임시 canvas 새로 생성 + `ctx.translate · rotate · scale · drawImage` 한 번에 적용. crop은 transformedCanvas 좌표계 기준이라 회전 후 자동 재정렬(`resetCropToFull`).

**비율 프리셋 7종**:
| ratio | 라벨 | 용도 |
|---|---|---|
| 0 | 자유 | 원하는 대로 |
| 1.0 | 1:1 | 인스타 정사각·프로필 |
| 1.3333 | 4:3 | 일반 카메라·구형 모니터 |
| 1.7778 | 16:9 | 유튜브·가로 영상 |
| 0.5625 | 9:16 | 인스타 릴스·틱톡·쇼츠 |
| 1.5 | 3:2 | DSLR·필름·프린트 4×6 |
| 0.8 | 4:5 | 인스타 세로 |

**8 핸들 인터랙션**:
- corner 4개 (tl·tr·bl·br) = 두 변 동시 변경. 비율 고정 시 wRatio·hRatio 중 큰 변화 기준으로 다른 변 동기화
- edge 4개 (tm·bm·lm·rm) = 한 변만 변경 (자유 비율). 비율 고정 시 다른 변을 가운데 기준 대칭 확장
- 안쪽 드래그 = 위치 이동 (크기 유지)
- bounds 제약 = 캔버스 안에 끼우기 + MIN_SIZE 16px + 비율 고정 후 bounds 잘렸을 때 비율 재계산

**Pointer Events 채택**: mousedown/move/up + touchstart/move/end 분리 대신 `pointerdown·pointermove·pointerup` 통합. `setPointerCapture`로 핸들 밖 이동 시에도 추적. `touch-action: none` CSS로 모바일 스크롤 차단.

**회전 6종**:
- ↻ 90° (시계방향, +90)
- ↺ 90° (반시계방향, +270)
- 180°
- ↔ 좌우 반전 (flipH 토글)
- ↕ 상하 반전 (flipV 토글)
- 원래대로 (모두 리셋)

**출력**: JPG (품질 슬라이더) / PNG (무손실) / WebP (효율 ↑). canvas → toBlob → BlobURL.

**SNS·플랫폼별 권장 비율 표** (tool-article inline) = 인스타 피드·릴스·스토리·유튜브·쇼츠·틱톡·트위터·페이스북·DSLR·프린트 4×6·5×7 11행.

## 신규 도구 3 — `image/id-photo/`

**vendor**: 없음 (canvas 2D API + Pointer Events)

**한국 표준 사이즈 6종 + 직접 입력**:
| 용도 | mm | 300 DPI 픽셀 |
|---|---|---|
| 여권 / 운전면허 / 주민등록증 | 35 × 45 | 413 × 531 |
| 일반 증명사진 / 이력서 / 반명함 | 30 × 40 | 354 × 472 |
| 미국 비자 (정사각) | 51 × 51 | 602 × 602 |
| 중국 비자 | 33 × 48 | 390 × 567 |
| 일본 비자 / 학생증 (참고) | 40 × 45 | 472 × 531 |
| 대형 명함 사진 (참고) | 45 × 60 | 531 × 709 |

**300 DPI 변환** = `MM_TO_PX = 300 / 25.4 ≈ 11.811`. 사진관·인쇄소 표준 해상도.

**가이드 라인** (frame 안 overlay):
- `.guide-h.top` (18%) = 정수리 권장 위치
- `.guide-h.eye` (42%) = 눈 라인
- `.guide-h.chin` (78%) = 턱끝
- `.guide-v` (50%) = 코·중심선
- 모두 점선 + 반투명 흰색 + 라벨 (정수리·눈·턱)

**비율 강제** = 프리셋이 `widthMm/heightMm` 결정. drag handle 4개 (corner만, edge X — 단순화). 비율 고정 시 핸들 동작은 crop과 동일 (큰 변화 기준).

**출력 — 표준 픽셀 리샘플링**:
```js
const out = new Canvas(targetW, targetH);  // 예: 413 × 531
ctx.imageSmoothingEnabled = true;
ctx.imageSmoothingQuality = 'high';
if (whiteBgOption) { ctx.fillStyle = '#ffffff'; ctx.fillRect(0, 0, targetW, targetH); }
ctx.drawImage(srcCanvas, crop.x, crop.y, crop.w, crop.h, 0, 0, targetW, targetH);
// canvas → toBlob('image/jpeg', 0.95) → BlobURL
```

**흰 배경 옵션** = PNG 알파 영역에 흰색 채우기. 원본이 단순 크롭이면 효과 없음 — 배경 제거 도구로 먼저 처리한 후 본 도구 사용 권장 (FAQ 명시).

**파일명** = `${stem}-35x45mm.jpg` 형태로 자동 생성.

**SEO 키워드 매핑**:
- title: "증명사진 만들기 | 여권·이력서·민증 사진 크기 무료 크롭"
- keywords: "증명사진 만들기, 여권 사진 크기, 이력서 사진, 민증 사진, 운전면허 사진 사이즈, 35x45 사진, 30x40 사진, 반명함 사진, 비자 사진"
- 출처 = 외교부 여권안내(passport.go.kr) · 도로교통공단(safedriving.or.kr) · 정부24 · 사진관 인쇄 표준 DPI 300

**비공식 용도 명시** (FAQ + sources note): 정부 신분증 발급용은 사진관 직접 촬영 권장 (여권은 6개월 이내·사진관 표시 등 요구). 본 도구는 **이력서·자기소개서·동아리·학원·앱 가입** 등 비공식 용도에 적합. 공식 신분증용은 발급 기관 가이드라인 직접 확인 안내.

- **2차 이미지 묶음 (보류)**:
  - **EXIF 제거** (~30KB piexifjs) — 프라이버시 키워드, "사진 EXIF 삭제·GPS 제거"

- **id-photo 흰배경 + 배경 제거 통합** — bg-remove의 출력을 id-photo로 자동 전달하는 워크플로우

## 학습 포인트

1. **esm.sh `?bundle` 함정** — 일부 패키지(libheif-js)는 의존성을 안 묶고 wrapper만 반환. 응답 사이즈가 비정상적으로 작으면(< 10KB) 의심. **해결책 = jsdelivr 빌드 산출물 직접 다운로드** (heic2any처럼 자체 번들된 라이브러리 선호)
2. **transformedCanvas 패턴** — 회전·반전 누적 적용 도구는 임시 canvas에 transform 후 drawImage로 한 번에. crop 좌표계는 transformedCanvas 기준이라 작업 일관됨
3. **Pointer Events 통일** — mouse + touch 분리 대신 `pointerdown/move/up`. `setPointerCapture`로 핸들 밖에서도 추적. `touch-action: none` CSS로 모바일 스크롤 충돌 방지
4. **비율 강제 corner 핸들** — 어떤 변이 더 변했는지(`Math.abs(nw - s.w)` vs `Math.abs(nh - s.h)`) 기준으로 다른 변 동기화 + 핸들 반대편 고정
5. **DPI 픽셀 변환** — 한국 사진 표준은 모두 mm 기반. 300 DPI 인쇄 표준 = `mm × 11.811`. 사진관·인쇄소 호환 픽셀 자동 산출이 차별화
6. **이미지 카테고리 펼치기 한계** — 9선이 한 hub에 적당. 12선+ 부풀면 favicon·EXIF 등 작업 시 카테고리 분리 검토 (이미 PDF는 분리됨, 다음은 "사진 편집"·"파일 변환" 등 분기 가능)
