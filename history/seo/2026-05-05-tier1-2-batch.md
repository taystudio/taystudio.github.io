## 부족했던 것 → 적용 fix 8개

### fix 1 — ItemList JSON-LD 카테고리 hub 누락 채움

이전 = image·video만 적용. tools·text 누락.

**`tools/index.html`** = 36 도구 ItemList:
```json
{
  "@type": "ItemList",
  "name": "실생활 계산기 36선",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "연봉 실수령액", "url": "..." },
    /* ... 36개 ... */
  ]
}
```

**`text/index.html`** = 3 도구 ItemList (counter·kbd-convert·sns-format).

**효과** = SERP "more from this site" 리치 결과 후보. 카테고리 hub로 진입한 사용자가 다른 도구 발견 가능성 ↑.

**카테고리별 lead 톤**:
- tools/ — "연봉·세금·부동산·임신·육아·건강까지 36가지 계산기 한곳에서"

### fix 4 — HowTo JSON-LD 4 도구

**적용**:
- `image/qr-gen/` — 4 step (입력·오류복구·생성·다운로드)
- `image/pdf-merge/` — 4 step (선택·정렬·합치기·다운로드)
- `image/img-to-pdf/` — 4 step (선택·정렬·옵션·생성)
- `video/compress/` — 6 step (선택·해상도·CRF·오디오·압축·다운로드)

**예시**:
```json
{
  "@type": "HowTo",
  "name": "QR 코드 만드는 방법",
  "totalTime": "PT30S",
```

**효과** = "How to" 리치 스니펫(번호 매기기 단계 카드) 노출 후보. CTR ↑.

**잔여** = bg-remove·video/trim는 후속 진입 (트래픽 데이터 후 결정).

### fix 6 — favicon.ico 285KB → 5.2KB (54배 감소)

**원인** = 이전에 `png-to-ico` CLI 사용. default가 16~256 multi-size 모두 packing해 큼.

**해결** = `to-ico` npm 패키지로 16x16 + 32x32 PNG 그대로 packing:
```bash
rsvg-convert -w 16 -h 16 favicon.svg -o /tmp/favicon-16.png
rsvg-convert -w 32 -h 32 favicon.svg -o /tmp/favicon-32.png
node -e '
  const toIco = require("to-ico");
  const fs = require("fs");
  const files = ["/tmp/favicon-16.png", "/tmp/favicon-32.png"].map(f => fs.readFileSync(f));
  toIco(files).then(buf => fs.writeFileSync("favicon.ico", buf));
'
```

`to-ico`는 `png-to-ico`와 달리 input PNG를 그대로 packing(자동 multi-size 추가 X). 임시 npm install·node_modules·package-lock 정리(commit 대상 X).

**효과** = 첫 페이지 fetch 시 ~280KB 절감. 모바일 LTE에서 0.5~1초 단축.
