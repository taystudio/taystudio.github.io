왜 banner만으로 부족했나

직전 commit의 lang banner는 헤더 위 sticky 좁은 bar(font-size 13px, padding 9px). 시선이 본문에 가 있는 외국인은 상단 bar를 못 보고 페이지 탐색·이탈. "한 번에 발견" 보장이 안 됨. → hero 영역(페이지 첫 화면 중앙)에 큰 카드로 박아 무조건 시야에 들어오게.

카드 디자인

  • 위치 = hero 안 (eyebrow + title + tagline 다음). 첫 화면 중앙
  • flex layout: 🌐 icon · 본문(title + sub) · actions(CTA + ×)
  • 배경 = 파란 linear-gradient 은은한 강조 (rgba(37,99,235,0.08) → rgba(99,102,241,0.06))
  • CTA = 큰 파란 버튼 + box-shadow + hover transform
  • 모바일 ≤640px = wrap, actions full width 분할

visibility 제어 (site-chrome.js)

  • 카드 markup은 정적 HTML이지만 default hidden 속성
  • setupLangWelcomeCard()에서 shouldShowLangBanner() 같은 조건 재사용 — 브라우저 mismatch + dismiss·lang-pref 미설정
  • 조건 만족 시 card.removeAttribute('hidden') → 자연스럽게 노출
  • CTA 클릭 → lang-pref 저장 (사용자 명시 선택 학습)
  • dismiss × 클릭 → lang-banner-dismissed 저장 + 카드 hidden + banner도 함께 제거 (UX 일관)

🐛 Descender clearance fix (사용자 신고)

"Useful things" 타이틀에서 "g" 아래 부분 잘림. 원인 = line-height: .96 + -webkit-background-clip: text 조합. line-height < 1이면 element box가 descender보다 짧고, 그 영역 밖은 background gradient 안 그려져서 transparent 처리.

.hub-title {
  ...
  line-height: .96;
  /* descender(g·j·p·q·y) clearance */
  padding-bottom: 0.18em;
  background: linear-gradient(...);
  -webkit-background-clip: text; background-clip: text;
  -webkit-text-fill-color: transparent;
}

한국판은 한글에 descender 없어서 영향 X — 영어판 en/index.html에만 적용.
