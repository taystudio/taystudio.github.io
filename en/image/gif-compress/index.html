What is gifsicle.wasm?
gifsicle is the standard GIF compression/editing tool, maintained since 1997 by Eddie Kohler (Harvard CS professor). It's a C library shipped in almost every Linux distribution. The WebAssembly port runs the same engine in your browser. The colors / lossy / optimize options are identical to the original gifsicle CLI.
Large files (50MB+)?
Technically possible, but wasm memory limits (~500MB on mobile, ~2GB on desktop Chrome) can stall large GIFs. First use video/to-gif to reduce fps and resolution, then compress here. Most huge GIFs come from 30+ fps at 1280×720+.
Why does strength 5 only get me ~50% smaller?
GIF is already LZW-compressed, so further compression has a ceiling. For bigger reductions: ① drop fps (10fps = ~50% smaller), ② drop resolution (640 → 320 = ~75% smaller), ③ trim frames. These three are best applied upstream with video/to-gif.
Does it work on mobile?
Yes (iOS Safari, Chrome, Firefox all support wasm). Memory limits make 10MB or smaller GIFs ideal on mobile; desktop Chrome handles 100MB. The first run downloads gifsicle.wasm (~200KB), then it's cached.

FAQ

Does the animation survive compression?
Yes. gifsicle.wasm preserves every frame of a multi-frame GIF while adjusting colors, lossy, and optimize. The standard image compressor uses canvas API and keeps only the first frame — this tool is GIF-specific so the animation stays intact.
What is the lossy option?
It slightly dithers pixel colors to boost LZW compression. Higher values yield smaller files but visible noise creeps in. Recommended 0~80 (visually lossless). 100~200 is fine for social and messengers with mild noise.
What happens when I reduce the color count?
GIF uses a 256-color palette. Reducing to 128 · 64 · 32 shrinks the file but flattens gradients and smooth color transitions. Logos · illustrations · single-color GIFs work fine at 64. Photographic GIFs should stay at 256.
Why doesn't the regular image compressor work for GIFs?
The image compressor uses canvas API's toBlob(), which doesn't support multi-frame. Upload a GIF and only the first frame is kept and re-encoded as JPG · PNG · WebP. To keep the animation, use this GIF compressor.
Will already-small GIFs get even smaller?
There's a floor. A GIF already under 100KB may only shrink by <10%. Large GIFs (1MB+) see the biggest gains (30~80%).
Can I batch-process multiple GIFs?
Single file at a time for now. Batch processing is planned. For many files, lock in the strength and run them one after another — it's fast.
