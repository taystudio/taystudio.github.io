@imgly/background-removal 라이브러리
@imgly/background-removal (IMG.LY, AGPL-3.0/Commercial). 본 도구는 GPL 호환 무료 사용 케이스이며, 모델·WASM은 staticimgly.com CDN에서 다운로드 후 브라우저에 캐시됩니다.
왜 staticimgly.com에서 모델을 받나요?
AI 모델(43MB+) + ONNX Runtime WebAssembly 자산을 repo에 직접 commit하면 git clone·deploy가 너무 무거워집니다. 그래서 OCR(Tesseract)과 동일한 패턴으로, 라이브러리만 vendoring(82KB) 하고 모델·WASM은 첫 사용 시 CDN에서 받아 IndexedDB·HTTP 캐시에 저장합니다. 사용자 이미지는 어떤 외부로도 전송되지 않습니다.
WebAssembly·SharedArrayBuffer 요구사항
AI 추론에 WASM과 (가능하면) SharedArrayBuffer가 필요합니다. 현대 Chrome·Firefox·Edge·Safari(15+)는 모두 지원. SharedArrayBuffer가 없으면 single-thread로 fallback되어 느려질 수 있습니다.
큰 사진 처리 팁
AI 추론은 입력 해상도 제곱에 비례해 시간·메모리 소모. 가로 1500~2000px가 sweet spot. 4000px+ 사진은 데스크톱 Chrome에서도 30초+ 걸리거나 OOM. 본 사이트의 이미지 리사이즈 도구로 미리 줄이고 처리하세요.
모델 정확도 더 높이고 싶다면
현재 small 모델(~43MB). medium(~80MB)·large(~180MB) 모델은 다운로드 부담 커서 우선 제공 안 함. 사용자 피드백·트래픽 봐서 옵션 토글 추가 검토. 추가 옵션 필요하시면 GitHub Issue·이메일.

자주 묻는 질문

이미지가 외부 서버로 전송되나요?
이미지는 전송되지 않습니다. AI 추론은 브라우저 안의 ONNX Runtime Web(WASM)에서 100% 로컬 수행됩니다. 단 모델 파일과 WASM 런타임은 첫 사용 시 staticimgly.com CDN에서 다운로드되며 브라우저 IndexedDB·HTTP 캐시에 저장됩니다(이후 offline 가능). 사이트 운영자도 이미지 내용을 볼 수 없습니다.
처음 실행할 때 왜 오래 걸리나요?
첫 실행 시 AI 모델(small 약 43MB) + ONNX 런타임 WASM 파일을 다운로드합니다. 이후 같은 브라우저에서는 캐시되어 즉시 시작됩니다. 모바일 데이터로는 첫 사용을 피하고 Wi-Fi 환경에서 한 번 실행해두는 것을 권장.
어떤 사진에 잘 작동하나요?
인물·반려동물·물건(상품 사진 등)에 일반적으로 잘 작동합니다. 배경과 피사체의 색·명도 대비가 클수록 결과가 깔끔. 머리카락·털 같은 디테일은 small 모델 한계로 약간 거칠 수 있습니다(medium·large 모델은 더 정확하지만 다운로드 부담 큼).
결과가 만족스럽지 않을 때
① 배경과 피사체 대비가 강한 사진 사용(흰 배경 추천) ② 피사체가 너무 작으면 미리 잘라서 가까이 ③ 그림자·반사가 심하면 결과에 잔재 가능 ④ 머리카락 등 미세 디테일 안 살면 medium 모델 옵션이 답이지만 80MB 다운로드 → 향후 옵션 추가 검토 ⑤ 가급적 가로 1500px 이하의 사진이 처리 빠름.
결과는 어떤 포맷으로 저장되나요?
투명 배경을 보존하기 위해 PNG로 다운로드됩니다. 투명 영역이 필요 없다면(예: 다른 단색 배경 위에 합성) 그래픽 도구에서 새 배경 위에 올린 후 JPG로 저장하면 됩니다. 본 사이트의 이미지 압축 도구로 PNG → JPG 변환도 가능.
최대 이미지 크기는?
이론적 제한은 없으나 모바일에서는 가로 2000px 이상 사진은 메모리 부족 가능. 데스크톱 Chrome 기준 4000px까지 무난. 큰 사진은 미리 이미지 리사이즈 도구로 가로 1500~2000px로 줄인 후 처리하면 빠르고 안전합니다.
