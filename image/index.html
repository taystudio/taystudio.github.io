— 브라우저 안에서만 처리

사진이 외부 서버로 전송되지 않습니다 — 이력서·신분증·민감 사진 안심.

TinyPNG·remove.bg 같은 도구는 클라우드 업로드 방식이지만, 본 사이트는 모든 처리가 사용자 브라우저 메모리·WebAssembly에서만 일어납니다. 이미지 압축·리사이즈·HEIC→JPG 변환(아이폰 사진)·자르기(SNS 비율)·증명사진(여권·이력서·민증)·QR 생성/인식·OCR·배경 제거(AI)·WebP/AVIF 변환·이미지 합치기·워터마크·모자이크/블러까지 14가지 이미지 도구를 무료·가입 없이 사용 가능합니다. 첫 사용 시 OCR(~13MB)·배경 제거(~43MB) AI 모델만 자동 다운로드되고 이후 캐시됩니다. PDF 합치기·자르기·편집·이미지↔PDF 변환은 별도 PDF 도구 카테고리로 분리되었습니다.

처리 방식·기술

누가 어떻게 쓰나
구직자·이력서증명사진(여권·이력서·민증 표준 사이즈 300DPI) 즉시 제작, 이력서 PDF에 들어갈 사진 압축, EXIF GPS 제거로 회사에 위치정보 노출 방지.
중고거래거래글 사진의 EXIF GPS·촬영일·기종 일괄 제거. 얼굴·번호판 모자이크/블러로 신원 노출 차단. WebP 변환으로 업로드 속도 향상.
인스타·블로그1:1·4:3·16:9·9:16 SNS 비율 프리셋 자르기, 카톡 캡처 9분할 합치기, 배경 제거(누끼)로 깔끔한 제품 사진, 워터마크로 도용 방지.
이미지 글자 추출스크린샷·사진·문서의 한국어·영어 텍스트 OCR 추출. 한자·중국어·일본어 옵션 지원. 영수증·메모·강의 노트 디지털화.
아이폰 사용자HEIC 사진을 윈도우·구형 갤러리에서 열 수 있게 JPG·PNG로 변환. 다중 페이지 HEIC도 자동 분리, 20장 이상 batch 처리 confirm.
디자이너·마케터WebP/AVIF로 웹 페이지 이미지 용량 50%+ 감소. QR 코드 PNG·SVG 즉시 생성. 14 도구 한 곳에서.
모든 14 도구의 처리 라이브러리
  • 압축·리사이즈·자르기·합치기·증명사진·모자이크·워터마크 — Canvas API (브라우저 내장)
  • HEIC → JPG — heic2any (Mozilla libheif WASM)
  • QR 생성 — qrcode-generator
  • QR 인식 — jsQR (카메라 스캔 + 이미지 디코드)
  • OCR (한국어·영어·한자·중국어·일본어) — Tesseract.js v5 WASM
  • 배경 제거 (AI) — img.ly Background Removal (ONNX Runtime Web)
  • WebP/AVIF 변환 — Canvas toBlob (브라우저 native encoder)
  • EXIF 제거 — piexifjs 분석 + Canvas re-encode (메타데이터 자동 제거)
모바일·iPad·iPhone에서 가능한가요?

iOS Safari 15+, iPadOS 15+, Android Chrome 90+에서 모두 정상 작동. 카메라 권한이 필요한 도구(QR 인식·OCR 일부)는 Safari가 권한을 요청하고, 거부 시에는 갤러리에서 이미지를 업로드해 처리합니다. iCloud·Google Photos 같은 클라우드 사진을 바로 불러올 때 권한이 만료되면 재선택 안내가 표시됩니다.

iOS Safari는 메모리 한계가 데스크톱보다 낮아 (~500MB) 50MB+ 이미지는 다중 처리 시 메모리 부족이 발생할 수 있습니다. 배경 제거 AI(~43MB)·OCR(~13MB)는 첫 사용 시만 다운로드 후 IndexedDB 캐시.

JPG·PNG·WebP·AVIF·HEIC 어떤 게 좋은가요?

JPG — 사진에 좋고, SNS·이메일 첨부 호환성 최고. 투명 배경 X.

PNG — 로고·아이콘·투명 배경 필요할 때. 사진은 용량 큼.

WebP — JPG보다 25~35% 작고 투명 배경 가능. 2020년 이후 모든 브라우저 지원.

AVIF — WebP보다 30% 더 작음. Safari 16+, Chrome 85+ 지원이지만 카톡·일부 구형 뷰어 호환성 떨어짐.

HEIC — iPhone 기본 포맷, 가장 작지만 윈도우·구형 갤러리 안 열림. 공유 전 JPG/PNG로 변환 권장.

자주 묻는 질문

업로드한 파일이 정말 외부로 안 가나요?
네. 모든 처리는 사용자 브라우저의 JavaScript·WebAssembly로만 일어납니다. 이미지 압축은 canvas API, PDF는 pdf-lib(JS), OCR은 Tesseract.js(WASM), 배경 제거는 ONNX Runtime Web(WASM)에서 클라이언트 사이드로 동작. 사이트 운영자도 사용자 파일 내용을 볼 수 없습니다. 첫 사용 시 OCR 학습 데이터·배경 제거 AI 모델만 jsdelivr·staticimgly.com에서 자동 다운로드됩니다.
한국어 OCR 정확도는 어느 정도인가요?
Tesseract.js의 한국어 학습 데이터 기준 인쇄체 명조·고딕은 90~95% 정확. 손글씨·필기체·복잡한 표·세로쓰기는 50~70%로 떨어집니다. 정확도는 이미지 해상도(300dpi+ 권장)·배경 대비·폰트 종류에 크게 좌우됩니다.
큰 파일도 처리 가능한가요?
데스크톱 Chrome 기준 100MB+ 이미지·PDF도 처리 가능. iOS Safari·모바일 브라우저는 메모리 한계로 ~500MB 이상에서 불안정할 수 있습니다. PDF 합치기처럼 여러 파일 동시 처리 시 합산 메모리 기준입니다.
AI 모델 다운로드는 매번 일어나나요?
아니요. 첫 사용 시만 다운로드되고 IndexedDB·브라우저 캐시에 저장돼 다음 방문은 즉시 시작됩니다. OCR 한국어 ~13MB, 배경 제거 small 모델 ~43MB. Wi-Fi 환경 권장.
