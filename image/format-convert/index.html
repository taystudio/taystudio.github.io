WebP·AVIF·JPG·PNG·GIF 정확히 뭔가요?
JPG(1992): 사진용 손실 압축, 범용 호환성. PNG(1996): 무손실·투명 지원, 로고·아이콘. GIF(1987): 256색 + 애니메이션, 옛 웹 그래픽. WebP(2010, Google): 같은 화질 JPG의 70~75% 용량, Chrome·Firefox·Safari 지원. AVIF(2019, AV1 기반): WebP보다 더 압축, 최신 브라우저만 지원.
왜 카톡·인스타에서 받은 사진이 webp로 되어있나요?
서버 비용·로딩 속도 때문입니다. 같은 화질이면 WebP가 JPG보다 25~35% 작아서 데이터·서버 비용이 절감됩니다. 사용자가 원본을 다운로드해도 메신저·SNS는 자동으로 WebP로 변환해서 줍니다. 받은 사람이 원본 JPG가 필요하면 본 도구로 변환.
한 번에 몇 장까지 변환되나요?
이론적으로 제한 없으나 브라우저 메모리에 좌우됩니다. 데스크톱 Chrome 기준 50~100장 무난(한 장당 평균 2~3MB 가정), 모바일 20~30장 권장. 큰 묶음은 한 번에 10~20장씩 나눠 처리.
EXIF(촬영 날짜·GPS) 정보는 보존되나요?
Canvas API 변환 과정에서 EXIF 메타데이터가 사라집니다. SNS·블로그 업로드 시엔 GPS 제거가 오히려 프라이버시에 유리하지만, 촬영 일시 보존이 필요하면 별도 EXIF 도구 사용 권장.

자주 묻는 질문

이미지 파일이 외부 서버로 업로드되나요?
전송되지 않습니다. 변환은 브라우저의 Canvas API와 native decoder로 모두 처리되며, 사이트 운영자도 사진 내용을 볼 수 없습니다.
왜 WebP가 안 열리나요?
WebP는 Google이 만든 새 이미지 포맷으로 같은 화질 기준 JPG보다 25~35% 작습니다. 카톡·인스타·네이버블로그가 자동으로 WebP로 저장하지만 한글 워드·PPT·일부 사진 뷰어·구버전 브라우저에서 안 열립니다. JPG로 변환하면 어디서나 열립니다.
AVIF는 무엇인가요?
AVIF는 AV1 코덱 기반의 차세대 이미지 포맷으로 WebP보다 더 압축됩니다. Chrome·Firefox·Safari 최신 버전에서 보이지만, Photoshop·한글·일부 뷰어에선 안 열립니다. 본 도구는 브라우저 native decoder로 AVIF를 읽어 JPG·PNG로 변환합니다.
JPG·PNG·WebP·AVIF 어느 것을 골라야 하나요?
JPG = 사진·인쇄·범용 호환성. PNG = 투명도·로고·아이콘·무손실. WebP = 웹 페이지 빠른 로딩·블로그 SEO. AVIF = 최신 웹 표준이지만 호환성 제한. 카톡·이메일로 보낼 땐 JPG, 디자인 작업 중이면 PNG, 본인 블로그·웹 게시면 WebP를 권장.
변환 시 화질이 떨어지나요?
JPG·WebP·AVIF는 손실 압축이라 품질 90 이상에서는 육안 차이 거의 없습니다. PNG는 무손실이라 화질 손상 없으나 용량이 큽니다. 본 도구는 원본을 디코딩한 픽셀 그대로 인코딩하므로 추가 손실은 없습니다.
여러 장을 한 번에 변환할 수 있나요?
네. 여러 파일을 드래그·드롭하거나 다중 선택해 한 번에 변환. 각 파일의 변환 진행률이 표시되며, 완료 후 페이지별 다운로드 또는 '전체 다운로드'로 한 번에 받을 수 있습니다.
