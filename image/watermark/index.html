회사 자료·보고서로고 + 우측 하단 + 투명도 60%

SNS 인용·아카이브텍스트(URL) + 좌측 하단 + 투명도 70%
위치·타일·투명도·회전 — 워터마크가 합성되는 원리
워터마크는 원본 픽셀 위에 텍스트나 로고를 Canvas로 덧그리는 합성입니다. 투명도는 globalAlpha로 0~1 사이 값을 줘 아래 사진이 비치게 합니다. 회전은 그릴 좌표로 translate한 뒤 rotate(도→라디안)로 캔버스를 돌려 기울인 글자를 만듭니다. 위치는 9칸 그리드(모서리·변·중앙)로, 각 칸은 가장자리에서 약 20px 안쪽 + 워터마크 크기 절반의 여백을 잡아 밖으로 잘리지 않게 좌표를 계산합니다. 반복(타일)을 켜면 워터마크 한 개 크기에 80px 간격을 더한 보폭으로 가로·세로를 훑으며 화면 전체를 덮습니다. 텍스트는 폰트 크기, 로고는 이미지 너비를 사진 너비의 비율(%)로 환산해 그립니다.
어디에 쓰나 — 저작권 표시 vs 유출 추적
쓰임은 크게 둘로 나뉩니다. 저작권·출처 표시는 닉네임·URL·로고를 눈에 띄되 사진을 가리지 않게(우측 하단, 투명도 50~70%) 넣어 “누구 것”인지 알립니다. 유출 억제·추적은 사진 전체를 가로지르는 반복 타일 + 회전 + 낮은 투명도(30~40%) 조합으로, 잘라내거나 지우기 어렵게 만들고 받는 사람·날짜를 함께 새겨 어디서 새어 나갔는지 단서를 남깁니다. 반복 타일은 잘라내기로 워터마크만 도려내려 해도 사진 본체까지 함께 잘려나가게 만드는 효과가 있어, 한 곳에만 찍는 것보다 억제력이 큽니다. 다만 워터마크는 복제를 100% 막는 장치가 아니라 비용·수고를 높여 억제하는 수단입니다. 가치가 큰 자산은 저해상도 공개나 저작권 등록(copyright.or.kr) 같은 추가 조치를 함께 권합니다.
9방향 위치·타일 간격·일괄 처리 — 실제 옵션과 좌표
위치는 9칸 그리드(좌상·상중·우상·좌중·중앙·우중·좌하·중하·우하)로 고르며, 각 칸은 워터마크가 가장자리에서 잘리지 않도록 워터마크 절반 크기 + 20px만큼 안쪽으로 들어간 좌표에 그려집니다(예: 우하는 가로 w − (워터마크폭/2 + 20)). 그래서 글자·로고가 커져도 모서리에서 잘려나가지 않습니다. 반복(타일)을 켜면 워터마크 한 개 크기에 80px 간격을 더한 보폭으로 가로·세로를 훑어 화면 전체를 덮습니다. 회전(−45°~+45°)은 도를 라디안으로 바꿔 적용하고, 투명도(10~100%)는 globalAlpha로 들어갑니다. 텍스트 크기는 px 절대값(12~120)으로, 로고 크기는 사진 너비 대비 비율(%)로 환산되므로 가로폭이 제각각인 여러 사진에 같은 비율로 일관되게 들어갑니다. 적용은 미리보기(첫 사진)로 위치를 잡은 뒤 누르면 올린 사진 전체에 같은 설정으로 일괄 처리되고, 진행바로 몇 장째인지 표시됩니다. 같은 파일명이 겹치면 _watermark-2처럼 번호를 붙여 덮어쓰기를 막고, 처리 도중 한 장이 인코딩에 실패해도 이미 성공한 장은 그대로 보존해 보여 줍니다. 출력은 JPG(품질 0.95)·PNG·WebP 중 고르며, JPG는 투명을 지원하지 않아 출력 전 흰 배경을 깔아 배경이 검게 나오는 것을 막습니다 — 투명 로고를 투명 배경째 남기려면 PNG를 선택하세요.

자주 묻는 질문

이미지가 외부 서버로 업로드되나요?
전송되지 않습니다. 모든 처리는 브라우저 Canvas API에서 이뤄집니다.
워터마크는 얼마나 강하게 도용을 막나요?
포토샵으로 크롭·블러·복원할 수는 있지만 시간이 들어 대부분의 도용을 막습니다. 강력한 보호는 (1) 반복 패턴(타일) (2) 사진 중앙 가로지름 (3) 적당한 투명도(40~60%) 조합. 본 도구는 모두 지원.
텍스트와 로고 이미지 둘 다 워터마크로 쓸 수 있나요?
네. 모드 전환으로 (1) 텍스트(폰트·크기·색상) (2) 로고 이미지(투명 PNG 권장) 중 선택.
여러 사진에 같은 워터마크를 한 번에 적용할 수 있나요?
네. 여러 파일을 드래그·드롭 후 워터마크 설정 1번만 → 일괄 적용. 미리보기로 위치 확인 가능.
원본 화질은 유지되나요?
원본 픽셀 그대로에 워터마크만 합성. JPG 출력 시 품질 95 권장. PNG 출력 시 무손실.
