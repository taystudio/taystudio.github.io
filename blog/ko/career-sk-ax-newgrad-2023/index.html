코딩테스트 + SKCT 심층 (2023-10-22)

A. 코딩테스트 (9:00–11:00, 2시간)

1번 — 구현 (모니터의 마우스 옮기기)

여러 대의 모니터 간 마우스 포인터 이동. 각 모니터의 좌하단을 (0,0) 기준. 마우스가 다른 모니터로 넘어가면 그 모니터의 (0,0) 부터 거리 측정.

N 개의 이동 명령 후 최종 위치 반환.

좌표계 변환 처리만 정확히 하면 무난.

2번 — 다이나믹 프로그래밍 (최솟값)

점화식 기반 최솟값 구하기.

점화식을 못 찾아서 스킵. — 끝나고 보니 비교적 표준적인 DP 였는데 그때는 못 봤음.

3번 — 구현 (격자 SLR 사이클)

격자에서 S (직진) · R (우회전 90°) · L (좌회전 90°) 명령으로 이동. 벽 충돌 시 180° 회전. 방문 칸 수 계산 + 무한 루프 감지.

무한 루프 감지가 핵심 — (x, y, dir) 상태를 Set 으로 관리해서 중복 시 break.

4번 — 누적합 (SQL)

한 달 일일 매출에서 누적 매출 SQL 로 구현.

윈도우 함수 SUM() OVER (ORDER BY date) 가 깔끔한 정답. 익숙하지 않았어서 self-join 으로 우회했는데 시간 많이 잡아먹음.

2시간 안에 4문제 진짜 빡셌음. SQL 누적합 도 직접 구현해본 적 없어서 헤맸음.

B. SKCT 심층 (13:00–14:30, 1.5시간)

SK 의 인성 검사 심층 버전. 일관되게 응답하면 됨 — "원래 본인이 하는 답" 이 맞고 일부러 좋게 꾸미면 일관성 점수에서 깎임.

결과: 코테 + SKCT 통과.

1day 면접 (2023-11-15)

토론 면접 + 직무/인성 면접. 반반씩 나뉘어 순서가 결정됨.

A. 토론 면접

8 명 → A 팀 4 + B 팀 4 로 나뉘어 상반된 입장에서 협상.

지표가 주어지고 점수를 더 따오는 것이 목표. 의견 조율 자체부터 모든 과정이 면접.

개인적으로 토론 면접은 진짜 재밌다고 생각했음. 시간 가는 줄 모르고 임함. 덕분에 긴장이 다 풀려서 다음 직무/인성 면접을 진짜 편하게 봄.

팁: - 본인 팀 안에서 역할 분담 미리 정리 (총괄·논거·중재·기록). - 상대 팀 의견을 무시하지 말고 인용 → 반박 형식. - 협상이 아니라 토론이라 결국 점수 따는 게 목적이라는 걸 잊지 말 것.

B. 직무/인성 면접

사전과제로 주어진 주제 → 2장 (표지 포함 3장) PPT 구성 → 약 10분 발표 → 면접관 질문.

  • 면접관: 3 인
  • 면접자: 2 인

직무/인성 면접에 대해서는 발표도 잘 했다 생각하고 본인의 소신을 거의 99% 잘 전달함. 후회 없는 면접이었고 어느 정도 붙을 거라 예상할 수 있었음.

팁: - 2 장 PPT — 결론 1 장 + 핵심 근거 1 장. 표지에는 본인 이름 + 제목만. - 10분 발표 → 5분 발표 + 5분 질문 시뮬. 발표 자체는 7분 안에 끝내는 게 안전. - 다른 면접자 답변도 적극 경청 → 본인 답변 차별화.

최종 결과

2023-12-05 최종 합격.

토론 면접에서 긴장이 풀린 게 결정적. 직무·인성 면접도 본인 소신을 명확하게 전달할 수 있었음. 결과적으로 면접 분위기 컨트롤 이 가장 중요한 요소 중 하나였다고 느낌.

정리 — SK 계열 신입 입장에서

  • 코테: 시뮬레이션 + DP + SQL. SQL 누적합 (윈도우 함수 또는 self-join) 미리 연습.
  • SKCT 심층: 일관된 본인 답이 정답. 꾸미면 들킴.
  • 토론 면접: 협상·토론 경험 없으면 모의 토론 1–2 회 해보고 가기. 본인 역할 미리 정하기.
  • 사전과제 PPT: 2장 — 결론 1 + 근거 1. 표지는 단순하게.
  • 직무/인성 면접: 본인 소신 99% 전달이 목표. 발표는 짧고 명확하게.

같은 길 준비하시는 분 건승.
