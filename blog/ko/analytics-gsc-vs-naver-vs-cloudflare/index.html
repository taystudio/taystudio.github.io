결론 먼저: GSC (Google Search Console) 데이터는 한국 사이트 진짜 트래픽의 10~20% 만 보여줌. 한국 사용자 대상 사이트면 Naver SearchAdvisor + Cloudflare Web Analytics 가 메인. GSC 는 보조.

사건 발단 — GSC 데이터에 패닉

신규 도메인 (taystudios.com) migration 1개월 차. GSC 열어보니:

3개월 데이터:
- 총 클릭: 2회
- 총 노출: 777회
- 평균 CTR: 0.3%
- 평균 게재 순위: 14.8

5/14 노출 정점 → 5/24 이후 거의 0. "사이트가 죽었나?"

근데 Cloudflare Web Analytics:

지난 24시간:
- Page views: 79
- Visits: 55

매일 55명 들어옴. 데이터 모순. 그리고 알게 된 진실 — 두 도구는 측정 대상이 완전 다름.

3종 도구의 본질 차이

도구 무엇을 측정? 데이터 출처
Google Search Console Google 검색 결과에서의 노출·클릭만 google.com → 우리 사이트 진입
Naver SearchAdvisor Naver 검색 결과 노출·클릭만 naver.com → 우리 사이트 진입
Cloudflare Web Analytics 사이트로 들어온 모든 트래픽 Direct·Bing·Naver·Daum·referral·소셜·LLM 다 포함

= GSC ↓ + Cloudflare 안정 = Google 검색 외 경로 트래픽 안정 의미. 모순 아님.

실제 데이터 — 같은 사이트, 한 달 비교

검색 노출 받는 키워드 Top 5:

  • "연봉 8000만원 실수령액" (노출 4, 순위 6.8)

  • "누끼 사이트" (노출 2, 순위 48.5)

발견 1 — Google 은 한국 사이트의 "20% 채널"

Naver: GSC 의 33~115배

Google 점유율이 한국 30~35% 인데도 우리 사이트는 Google 트래픽이 더 작음. 이유:

a. Google sandbox (신규 도메인 6-12주 평가)

도메인 migration 5/9 → 6/2 = 3.5주 차. 색인 122개 됐지만 검색 순위 상승 제한 상태.

b. AI Overview 의 zero-click

"연봉 5000만 실수령액" 검색 시 Google 이 직접 AI 답 → 우리 사이트 클릭 X. 한국 정책 데이터의 AI 학습이 Naver 보다 Google 이 더 강함 (Gemini).

c. 한국 키워드 NLP 차이

"육아휴직 급여 계산기" 같은 한국 정책 키워드 → Naver 가 더 정확하게 매칭. Google 의 한국어 NLP 는 Naver 자체 모델보다 약함.

→ GSC 만 보면 우리 사이트의 80% 채널을 못 본다.

발견 2 — Cloudflare 의 핵심 데이터는 Visits Detail

Cloudflare Web Analytics 의 진짜 정보는 메인 화면이 아니라 Visits 카드 클릭 후 detail 화면:

Visits by source (Referrers):
- m.search.naver.com
- search.naver.com
- search.daum.net
- chatgpt.com

Hosts:
- taystudios.com

Top paths:
- /tools/baby-formula/: 16
- /tools/salary/: ...
- /image/bg-remove/: ...

Browsers, OS, Country, Device types

이 detail 보면 진짜 출처 보임. GSC "Google 검색 0건" 라도 Naver mobile + desktop + Daum 합쳐 매일 50+ 명 들어옴.

특히 chatgpt.com referrer = AI 검색에서 사이트 인용 시작의 신호. GSC·Naver 둘 다 안 보여줌.

발견 3 — Top 페이지 mismatch

측정 Top 페이지
GSC Top 키워드 → 페이지 salary·property-tax·cartax·bg-remove
Cloudflare Top 방문 페이지 /tools/baby-formula/

baby-formula 가 GSC 에 안 보이는 페이지인데 Cloudflare 1위. 이유: Naver "분유량" 검색 매칭. Naver 트래픽이 GSC 에 안 잡혀서 baby-formula 가 Google 데이터에서 invisible.

→ 진짜 인기 페이지 알려면 Cloudflare 봐야 함. Google 데이터는 빙산의 일각.

3 도구의 올바른 사용법

GSC — Google 평가 + 키워드 발견용

  • 어떤 키워드로 노출 받는지 (search query)
  • 색인 상태 (몇 페이지 색인 됐는지)
  • 평균 순위
  • 페이지별 색인 사유 (안 된 페이지 진단)
  • traffic 절대량 측정 X — 한국 사이트는 빙산의 일각

Naver SearchAdvisor — 한국 사용자 메인 채널

  • 한국 사용자 대상 사이트면 가장 중요
  • sitemap 제출·웹페이지 수집 요청 가능
  • 노출·클릭 현황 (한국 트래픽 거의 전부 포함)
  • robots·웹페이지 진단
  • 등록 안 하면 자체 크롤로 우연히 발견 — 매우 느림

Cloudflare Web Analytics — 진짜 트래픽 + 출처

  • 모든 채널 합산 (Direct + 검색엔진 + referral)
  • Sources detail = 어디서 들어왔는지
  • Top paths = 진짜 인기 페이지
  • 국가·기기·브라우저 breakdown
  • LLM referrer (chatgpt.com·perplexity.ai) 보이기 시작
  • 개인정보 추적 없음 (Privacy-friendly)

우리 액션 우선순위

3종 데이터 통합 결과:

  1. Naver Webmaster 등록 + sitemap 제출 — 가장 큰 ROI (트래픽 79% 채널)
  2. GSC URL 검사로 변경 페이지 수동 색인 요청 — sandbox 영향 일부 우회
  3. Bing Webmaster (GSC 1-click import 가능) — 추가 1-3%
  4. Daum Webmaster — 5~10% 추가
  5. Cloudflare Sources 정기 점검 — LLM referrer 추세 확인

한 줄 결론

한국 사이트는 GSC 만 보면 80% 데이터 놓침. Naver + Cloudflare 동시 보기.

GSC 는 "Google 검색 데이터" 만. 한국 사이트의 진짜 트래픽은 한국 검색엔진 + AI 검색 + Direct 가 메인. 하나의 데이터로 판단하면 큰 그림 못 봄. 신규 도메인 운영자가 가장 흔히 빠지는 함정.
