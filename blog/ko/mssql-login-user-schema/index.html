1. м„ңлЎ 

мөңк·ј CI/CD л°°нҸ¬ кҙҖлҰ¬ мӢңмҠӨн…ң кө¬м¶• н”„лЎңм қнҠём—җм„ң TA м—ӯн• мқ„ мҲҳн–үн•ҳл©° мқјл¶Җ DBA м—…л¬ҙк№Ңм§Җ л§ЎкІҢ лҗҳм—Ҳкі , к·ё кіјм •м—җм„ң MS SQL Serverмқҳ кі„м • кҙҖлҰ¬ кө¬мЎ°лҘј м§Ғм ‘ лӢӨлЈЁкІҢ лҗҗлӢӨ.

мқҙм „м—җ Oracle DBлҘј лӢӨлӨ„ліё кІҪн—ҳмқҙ мһҲм—Ҳм§Җл§Ң, л‘җ DBMSлҠ” кі„м •(User)В·мҠӨнӮӨл§Ҳ(Schema)В·к¶Ңн•ң кҙҖлҰ¬ к°ңл…җмқҙ нҒ¬кІҢ лӢ¬лқј мҙҲл°ҳм—җ мғҒлӢ№н•ң нҳјлһҖмқ„ кІӘм—ҲлӢӨ. мқҙ кёҖм—җм„ңлҠ” Oracleм—җм„ң MS SQL ServerлЎң л„ҳм–ҙмҳ¬ л•Ң нқ”нһҲ н—·к°ҲлҰ¬лҠ” кі„м •В·мҠӨнӮӨл§ҲВ·к¶Ңн•ң кө¬мЎ°мқҳ м°ЁмқҙлҘј м •лҰ¬н•ҳкі , м§Ғм ‘ кІҪн—ҳн•ң sysadmin кі„м •мқҳ dbo мҠӨнӮӨл§Ҳ кі м • л¬ём ңмҷҖ кө¬мЎ°м Ғ н•ҙкІ° л°©лІ•к№Ңм§Җ н•Ёк»ҳ кіөмң н•ңлӢӨ.

2. Oracleкіј MS SQL Serverмқҳ кі„м •В·мҠӨнӮӨл§Ҳ к°ңл…җ м°Ёмқҙ

л§ҺмқҖ Oracle кІҪн—ҳмһҗк°Җ SQL ServerлҘј мІҳмқҢ м ‘н•ҳл©ҙ к°ҖмһҘ н—·к°Ҳл Өн•ҳлҠ” л¶Җл¶„мқҙ л°”лЎң лЎңк·ёмқё кі„м •кіј мҠӨнӮӨл§Ҳк°Җ л”°лЎң л…јлӢӨлҠ” м җмқҙлӢӨ.

Oracle вҖ” User = Schema = Login

  • кі„м •мқ„ мғқм„ұн•ҳл©ҙ мһҗлҸҷмңјлЎң лҸҷмқјн•ң мқҙлҰ„мқҳ мҠӨнӮӨл§Ҳк°Җ мғқкёҙлӢӨ.
  • мҰү SCOTT кі„м •мқ„ л§Ңл“Өл©ҙ кі§л°”лЎң SCOTT мҠӨнӮӨл§Ҳм—җм„ң мһ‘м—…н•ңлӢӨ.
CREATE USER SCOTT IDENTIFIED BY tiger;
-- лҸҷмӢңм—җ SCOTT мҠӨнӮӨл§Ҳ мғқм„ұ

MS SQL Server вҖ” Login вҶ’ User вҶ’ Schema (3лӢЁкі„)

  • Login кі„м • (м„ңлІ„ л ҲлІЁ) вҖ” SQL Server мқёмҠӨн„ҙмҠӨ м ‘мҶҚмҡ© кі„м •. м„ңлІ„ м „мІҙ кҙҖм җмқҳ мқёмҰқ кі„м •мңјлЎң, ID/PWлЎң мқёмҠӨн„ҙмҠӨм—җ лЎңк·ёмқён•ңлӢӨ.
  • User кі„м • (DB л ҲлІЁ) вҖ” лЎңк·ёмқё мқҙнӣ„ нҠ№м • лҚ°мқҙн„°лІ мқҙмҠӨм—җ мҶҢмҶҚлҗҳлҠ” мӮ¬мҡ©мһҗ кі„м •. н•ҳлӮҳмқҳ LoginмқҖ к°Ғ лҚ°мқҙн„°лІ мқҙмҠӨ лӮҙмқҳ UserмҷҖ 1:1лЎң л§Өн•‘лҗҳм–ҙм•ј н•ңлӢӨ. мҰү лҸҷмқјн•ң LoginмқҙлқјлҸ„ DBл§ҲлӢӨ лі„лҸ„мқҳ UserлҘј к°Җм§Ҳ мҲҳ мһҲлӢӨ.
  • Schema (к°қмІҙ м»Ён…Ңмқҙл„Ҳ) вҖ” н…Ңмқҙлё”В·л·°В·н”„лЎңмӢңм Җ л“ұ DB к°қмІҙлҘј л…јлҰ¬м ҒмңјлЎң кө¬л¶„н•ҳлҠ” м»Ён…Ңмқҙл„Ҳ. UserмҷҖлҠ” лі„лҸ„мқҳ к°ңл…җмқҙл©°, к°Ғ UserлҠ” мӮ¬мҡ©н•  кё°ліё мҠӨнӮӨл§Ҳ(Default Schema)лҘј м§Җм •н•  мҲҳ мһҲлӢӨ. мІҳмқҢ кё°ліёк°’мқҖ dbo мҠӨнӮӨл§Ҳмқҙл©°, мқҙлҘј мӢӨм ң мӮ¬мҡ©н•  мҠӨнӮӨл§ҲлЎң л°”кҝ”мӨҳм•ј н•ңлӢӨ.
-- 1) м„ңлІ„ лЎңк·ёмқё кі„м • мғқм„ұ
CREATE LOGIN SCOTT WITH PASSWORD = '***';

-- 2) Database мғқм„ұ
CREATE DATABASE SCOTT_tiger;

-- 3) мҠӨнӮӨл§Ҳ мғқм„ұ
CREATE SCHEMA vdadmin AUTHORIZATION vdadmin_user;

-- 4) нҠ№м • DBм—җ мӮ¬мҡ©мһҗ мғқм„ұ
--    MS SQLмқҖ н•ӯмғҒ нҠ№м • DBлҘј мӮ¬мҡ©н•ңлӢӨкі  мҠӨмң„м№ҳ(USE)н•ҙмӨҳм•ј н•ңлӢӨ.
--    (OracleмқҖ schemaмҷҖ кі„м •мқҙ 1:1мқҙлқј к·ёлҹҙ н•„мҡ”к°Җ м—ҶлӢӨ)
USE SCOTT_tiger;
-- SSMSм—җм„ңлҠ” GO лӘ…л №м–ҙк№Ңм§Җ м Ғм–ҙмӨҖлӢӨ (DBeaver кё°мӨҖ м•Ҳ м Ғм–ҙлҸ„ лҗЁ)
CREATE USER db_user FOR LOGIN SCOTT WITH DEFAULT_SCHEMA = SCOTT_tiger;
-- SCOTT_tiger DB м•Ҳм—җ db_user кі„м •мқ„ лЎңк·ёмқё кі„м • SCOTTкіј л§Өн•‘н•ҙ л§Ңл“Өкі ,
-- н•ҙлӢ№ db_userмқҳ л””нҸҙнҠё мҠӨнӮӨл§ҲлЎң SCOTT_tigerлҘј м§Җм •н•ңлӢӨ.

м •лҰ¬н•ҳл©ҙ, MS SQL ServerлҠ” 3лӢЁкі„ кө¬мЎ°(Login вҶ’ User вҶ’ Schema)лӢӨ. м„ңлІ„ л ҲлІЁмқҳ Loginмқҙ к°Ғ DB лӮҙмқҳ UserмҷҖ л§Өн•‘лҗҳкі , к·ё Userк°Җ DB лӮҙ м—¬лҹ¬ Schema мӨ‘ н•ҳлӮҳлҘј кё°ліё мҠӨнӮӨл§ҲлЎң м§Җм •н•ҙ мӮ¬мҡ©н•ңлӢӨ. л°ҳл©ҙ OracleмқҖ 1лӢЁкі„ кө¬мЎ°(User = Schema = Login)лЎң, лЎңк·ёмқё кі„м •мқҙ кі§ мӮ¬мҡ©мһҗмқҙмһҗ мҠӨнӮӨл§ҲлЎң мһ‘лҸҷн•ҳл©° лі„лҸ„мқҳ л§Өн•‘ к°ңл…җмқҙ м—ҶлӢӨ.

рҹ“ҳ м°ёкі  вҖ” MS SQLм—җм„ң Default SchemaлҘј м§Җм •н•ҳлҚ”лқјлҸ„ лӢӨлҘё мҠӨнӮӨл§Ҳмқҳ к°қмІҙлҘј лӘ» м“°лҠ” кІғмқҖ м•„лӢҲлӢӨ. schema.table нҳ•мӢқмңјлЎң лӘ…мӢңн•ҳл©ҙ лӢӨлҘё мҠӨнӮӨл§Ҳмқҳ н…Ңмқҙлё”м—җлҸ„ мһҗмң лЎӯкІҢ м ‘к·јн•ҙ мҝјлҰ¬н•  мҲҳ мһҲлӢӨ.

лӢӨл§Ң мҷёл¶Җ мӢңмҠӨн…ң м—°лҸҷмқҙлӮҳ м• н”ҢлҰ¬мјҖмқҙм…ҳ м—°кІ° кі„м •мқ„ кө¬м„ұн•  л•ҢлҠ” Default SchemaлҘј лӘ…нҷ•нһҲ м§Җм •н•ҙл‘җлҠ” кІғмқҙ мӨ‘мҡ”н•ҳлӢӨ. к·ёл Үм§Җ м•Ҡмңјл©ҙ м—°кІ° мӢң dbo лҳҗлҠ” м§Җм •лҗҳм§Җ м•ҠмқҖ кё°ліё мҠӨнӮӨл§ҲлЎң мһҗлҸҷ л§Өн•‘лҗҳм–ҙ мқҳлҸ„мҷҖ лӢӨлҘё к°қмІҙлҘј м°ёмЎ°н•ҳлҠ” л¬ём ңк°Җ мғқкёё мҲҳ мһҲлӢӨ. л”°лқјм„ң Login вҶ’ User вҶ’ Default Schema кҙҖкі„лҘј лӘ…нҷ•нһҲ л§Өн•‘н•ҳкі , мҠӨнӮӨл§Ҳлі„лЎң м „мҡ© лЎңк·ёмқё кі„м •мқ„ л§Ңл“Өм–ҙ н•ҙлӢ№ мҠӨнӮӨл§Ҳл§Ң м“°лҸ„лЎқ кҙҖлҰ¬н•ҳлҠ” кІғмқҙ к°ҖмһҘ м•Ҳм „н•ҳкі  нҡЁмңЁм ҒмқҙлӢӨ. (кІ°көӯ Oracleмқҳ лӢЁмқј кө¬мЎ°к°Җ нӣЁм”¬ м§ҒкҙҖм Ғмқҙкёҙ н•ҳлӢӨ.)

3. Default Schemaмқҳ м°Ёмқҙ

Oracle

  • кі„м •мңјлЎң м ‘мҶҚн•ҳл©ҙ н•ҙлӢ№ кі„м •мқҳ мҠӨнӮӨл§Ҳк°Җ мһҗлҸҷмңјлЎң Default Schemaк°Җ лҗңлӢӨ.
  • User = Schema кө¬мЎ°лқј лі„лҸ„ м„Өм •мқҙ н•„мҡ” м—ҶлӢӨ. SCOTTмңјлЎң лЎңк·ёмқён•ҳл©ҙ мһҗлҸҷмңјлЎң SCOTT мҠӨнӮӨл§Ҳм—җ м—°кІ°лҗңлӢӨ.

SQL Server

  • User мғқм„ұ мӢң DEFAULT_SCHEMAлҘј м§Җм •н•ҳм§Җ м•Ҡмңјл©ҙ кё°ліёк°’мқҖ н•ӯмғҒ dboлӢӨ.
  • мӣҗн•ҳлҠ” мҠӨнӮӨл§ҲлҘј кё°ліёмңјлЎң м“°л Өл©ҙ лӘ…мӢңм ҒмңјлЎң м§Җм •н•ҙм•ј н•ңлӢӨ.
ALTER USER a_user WITH DEFAULT_SCHEMA = a_schema;

мқҙл ҮкІҢ м„Өм •н•ҳм§Җ м•Ҡмңјл©ҙ к°қмІҙ мғқм„ұ мӢң dbo.table_name нҳ•нғңлЎң мһҗлҸҷ мғқм„ұлҗҳм–ҙ, мқҳлҸ„мҷҖ лӢӨлҘҙкІҢ dbo мҠӨнӮӨл§Ҳ м•„лһҳм—җ к°қмІҙк°Җ мҢ“мқҙлҠ” л¬ём ңк°Җ л°ңмғқн•  мҲҳ мһҲлӢӨ.

4. к¶Ңн•ң кҙҖлҰ¬ л№„көҗ (Oracle vs SQL Server)

л‘җ DBMS лӘЁл‘җ м ‘к·јВ·мһ‘м—… м ңм–ҙлҘј мң„н•ҙ к¶Ңн•ңкіј м—ӯн• (Role)мқ„ м ңкіөн•ҳм§Җл§Ң, кө¬мЎ°мҷҖ м Ғмҡ© лІ”мң„к°Җ лӢӨлҘҙлӢӨ.

Oracle вҖ” к¶Ңн•ңмқҙ нҒ¬кІҢ System к¶Ңн•ңкіј Object к¶Ңн•ңмңјлЎң лӮҳлүңлӢӨ.

  • System к¶Ңн•ң вҖ” DB м „мІҙм—җ мҳҒн–Ҙмқ„ лҜём№ҳлҠ” к¶Ңн•ң (мҳҲ: CREATE SESSION, CREATE TABLE).
  • Object к¶Ңн•ң вҖ” нҠ№м • н…Ңмқҙлё”В·л·° л“ұ к°ңлі„ к°қмІҙм—җ лҢҖн•ң м ‘к·ј к¶Ңн•ң (SELECT, INSERT, UPDATE, DELETE).
  • м—¬лҹ¬ к¶Ңн•ңмқ„ л¬¶лҠ” Roleмқ„ м ңкіөн•ҙ DBA, RESOURCE, CONNECT к°ҷмқҖ мӮ¬м „ м •мқҳ м—ӯн• мқ„ нҷңмҡ©н•ҳкұ°лӮҳ мӮ¬мҡ©мһҗ м •мқҳ м—ӯн• мқ„ л§Ңл“Ө мҲҳ мһҲлӢӨ.

SQL Server вҖ” к¶Ңн•ң мІҙкі„к°Җ лҚ” кі„мёөнҷ”лҗҳм–ҙ мһҲлӢӨ.

  • м„ңлІ„ л ҲлІЁ к¶Ңн•ң вҖ” sysadmin, serveradmin, securityadmin л“ұ м„ңлІ„ м „мІҙм—җ мҳҒн–Ҙ.
  • DB л ҲлІЁ к¶Ңн•ң вҖ” CONNECT, CONTROL, ALTER л“ұ лҚ°мқҙн„°лІ мқҙмҠӨ лӢЁмң„.
  • мҠӨнӮӨл§Ҳ/к°қмІҙ л ҲлІЁ к¶Ңн•ң вҖ” нҠ№м • мҠӨнӮӨл§ҲлӮҳ к°қмІҙм—җ к¶Ңн•ң л¶Җм—¬.
GRANT SELECT, INSERT, UPDATE, DELETE ON SCHEMA::X TO user;
  • SQL ServerлҸ„ db_datareader, db_datawriter, db_ddladmin л“ұ кё°ліё м ңкіө RoleлЎң к¶Ңн•ңмқ„ л¬¶м–ҙ кҙҖлҰ¬н•  мҲҳ мһҲлӢӨ.

м •лҰ¬н•ҳл©ҙ, OracleмқҖ к¶Ңн•ңмқ„ мӢңмҠӨн…ң м „мІҙ vs к°қмІҙ лӢЁмң„лЎң кө¬л¶„н•ҳкі  Role мӨ‘мӢ¬мңјлЎң кҙҖлҰ¬н•ҳлҠ” л°ҳл©ҙ, SQL ServerлҠ” м„ңлІ„ вҶ’ DB вҶ’ мҠӨнӮӨл§Ҳ/к°қмІҙ мҲҳмӨҖмңјлЎң кі„мёөнҷ”лҗң к¶Ңн•ң мІҙкі„лҘј к°Җм§Җл©° м—ӯн• лҸ„ мқҙ кі„мёөкіј кІ°н•©лҗҳм–ҙ мһҲлӢӨ.

5. мӢӨл¬ҙм—җм„ң кІӘлҠ” нҳјлҸҷ нҸ¬мқёнҠё

sysadmin кі„м •мңјлЎң MS SQLм—җ лЎңк·ёмқён–Ҳмқ„ л•Ң, м•„л¬ҙлҰ¬ Default SchemaлҘј ліҖкІҪн•ҙлҸ„ н•ӯмғҒ dbo мҠӨнӮӨл§ҲлЎң л“Өм–ҙк°ҖлҠ” нҳ„мғҒмқ„ кІӘм—ҲлӢӨ. мқҙмң лҠ” лӘ…нҷ•н•ҳлӢӨ. MS SQLм—җм„ң sysadmin кі„м •мқҖ н•ӯмғҒ dbo мҠӨнӮӨл§Ҳм—җ л§Өн•‘лҗҳлҸ„лЎқ м„Өкі„лҗҳм–ҙ мһҲмңјл©°, мқҙлҠ” ліҙм•ҲВ·к¶Ңн•ң кҙҖлҰ¬ м •мұ…мғҒ кі м •лҗң лҸҷмһ‘мқҙлӢӨ.

мҰү DBA(sysadmin) кі„м •мқҖ Default SchemaлҘј мһҗмң лЎӯкІҢ л°”кҝҖ мҲҳ м—Ҷкі  н•ӯмғҒ dboм—җм„ң мӢңмһ‘н•ңлӢӨ. л”°лқјм„ң мӢӨл¬ҙм—җм„ңлҠ” sysadminкіј лі„лҸ„лЎң к°Ғ мҠӨнӮӨл§Ҳлі„ лЎңк·ёмқё м•„мқҙл””лҘј л§Ңл“Өм–ҙ л§Өн•‘н•ҙ мӮ¬мҡ©н•ҳлҠ” кІғмқҙ мқјл°ҳм Ғмқҙл©°, мқҙл ҮкІҢ н•ҳл©ҙ OracleмІҳлҹј кі„м •лі„ мҠӨнӮӨл§ҲлҘј мҡҙмҳҒн•  мҲҳ мһҲлӢӨ.

6. н•ҙкІ° л°©лІ• вҖ” м—…л¬ҙмҡ© кі„м •кіј мҠӨнӮӨл§Ҳ л¶„лҰ¬

DBA к¶Ңн•ң кі„м •мқҖ мҡҙмҳҒВ·кҙҖлҰ¬ м „мҡ©мңјлЎң л‘җкі , нҠ№м • мҠӨнӮӨл§ҲлҘј л””нҸҙнҠёлЎң м“°л Өл©ҙ лі„лҸ„мқҳ лЎңк·ёмқё кі„м •мқ„ л§Ңл“Өм–ҙм•ј н•ңлӢӨ.

  • м„ңлІ„ кҙҖлҰ¬мҡ© кі„м • вҖ” sysadmin (dbo мҠӨнӮӨл§Ҳ кі м •)
