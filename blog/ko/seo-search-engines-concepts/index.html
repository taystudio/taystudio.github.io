결론 먼저: 신규 도메인은 6-12주 정도 Google sandbox 에 들어가 있어서 노출이 거의 0 에 가깝습니다. 그동안 한국 사이트라면 Naver 가 사실상 본채널인데, GSC 만 들여다보면 사이트가 죽은 줄로 착각하기 쉽습니다. 거기에 AI Overview·zero-click 까지 겹치면서 클릭률은 더 떨어지는 추세고요. 제가 직접 운영하면서 느낀 건, SEO 는 80% 가 시간이고 액션으로 바꿀 수 있는 건 나머지 20% 정도라는 점입니다.

도메인을 옮기고(taystudio.github.io → taystudios.com) 한 달쯤 됐을 때입니다. GSC 를 열어 보니 클릭 2건에 노출 777. 솔직히 그 숫자만 보고 사이트가 잘못된 줄 알고 한참 당황했습니다. 그런데 Cloudflare 를 보니 일 방문이 55명으로 멀쩡했고, Naver SearchAdvisor 까지 등록해서 보니 한 달 노출이 26,000 에 클릭 230 이 찍혀 있었습니다.

결국 데이터의 한쪽 면만 보고 혼자 놀랐던 셈입니다.

이 글은 그 한 달 동안 제가 직접 부딪혀 가며 정리한 SEO 핵심 개념 12개입니다. 신규 도메인 운영자가 가장 흔히 빠지는 함정과, 한국 시장 특유의 사정까지 같이 다뤄 보겠습니다.

검색엔진 기본 흐름

사이트 → [크롤링] → [색인] → [순위 결정] → 검색 결과 노출

이 단계 중 어느 한 곳이라도 막히면 트래픽은 0 이 됩니다. 저희도 5/9 migration 이후 5/22 까지는 색인이 거의 없다가, 5/22 부터 본격적으로 색인이 잡히기 시작했습니다. 대략 13일이 걸린 셈입니다. 경험상 신규 도메인은 색인이 잡히는 것 자체에 시간이 걸린다고 보면 됩니다.

용어 의미
크롤링 (Crawling) 검색엔진 봇이 HTML 자동 방문해서 읽기. Googlebot·Naver Yeti·Bingbot
색인 (Indexing) 읽은 페이지를 검색엔진 DB 에 저장. 색인 없으면 검색 결과 안 나옴
순위 결정 (Ranking) 검색어와 색인 페이지 매칭 + 알고리즘으로 1~10000위 결정
노출 (Impression) 검색 결과 페이지에 우리 사이트가 표시된 횟수
클릭 (Click) 검색 결과에서 사이트로 진입한 횟수
CTR (Click-Through Rate) 클릭 ÷ 노출 × 100. 정상 2-5%, 우리 GSC 0.3% / Naver 0.9%

1. Google Sandbox — 신규 도메인 "견습 기간"

Google 이 신규 도메인을 일정 기간 일종의 "견습" 상태로 두는 비공식 알고리즘입니다. 보통 6-12주 동안은 색인은 되지만 검색 순위가 올라가는 것은 제한됩니다.

왜 존재하나

스팸이나 MFA(Made For AdSense) 사이트가 신규 도메인을 양산해서 노출을 받아가는 것을 막기 위한 장치라고 보면 됩니다. "이 사이트가 6주 뒤에도 살아있을 사이트인가" 를 지켜보는 시간인 셈입니다.

이런 신호가 나타납니다

  • 색인은 잘 됩니다 ✅
  • 노출이 올랐다가 다시 가라앉는 패턴 (저희는 5/14 정점 → 5/24 거의 0)
  • CTR 이 매우 낮음
  • 같은 키워드에서 오래된 도메인 사이트가 우선 노출됨

대응

결국 시간 + 콘텐츠 품질 유지 + 백링크 자연 발생, 이 세 가지입니다. 외부 액션으로 기간을 단축하는 것은 거의 불가능하다고 봅니다.

Google 은 공식적으로 sandbox 의 존재를 부정하지만, SEO 업계에서는 폭넓게 관찰되는 현상입니다. 저희 도메인은 5/9 시작이니 여기에 6-12주를 더하면 대략 6/20 ~ 7/30 사이에 자동으로 풀릴 것으로 예상하고 있습니다. 그때까지는 그냥 기다리는 게 정답이더군요.

2. E-E-A-T — Google 의 4 품질 기준

Experience · Expertise · Authoritativeness · Trustworthiness

기준 의미
Experience (경험) 실제 사용해본 사람이 쓴 글인가
Expertise (전문성) 작성자가 그 분야 전문가인가
Authoritativeness (권위) 이 사이트가 그 주제로 알려졌나
Trustworthiness (신뢰) 정확·투명·안전한가

특히 YMYL (Your Money, Your Life), 즉 세금·의료·법률·금융처럼 사람 인생에 영향이 큰 콘텐츠는 E-E-A-T 평가가 굉장히 엄격합니다. 잘못된 정보 한 건만으로도 사이트 전체의 신뢰도가 깎일 수 있습니다.

저희 사이트의 계산기 38개 중 절반 이상이 YMYL(salary·property-tax·inheritance 등)이라, 모든 수치는 정부 공식 사이트(law.go.kr·국세청·기재부)에서 직접 인용하는 것으로 통일했습니다. 블로그나 언론 인용은 한 건도 두지 않았습니다. E-E-A-T 위반 risk 를 처음부터 차단하려는 의도였습니다.

3. Domain Authority (DA)

사이트가 검색엔진에서 얼마나 신뢰받는지를 0~100 점수로 나타낸 지표입니다. Moz·Ahrefs 같은 곳이 측정하는데, Google 의 공식 점수는 아니고 어디까지나 추정치입니다.

이 점수를 가르는 핵심 요인은 백링크, 즉 다른 사이트가 우리를 링크해 주는 양과 질입니다.

  • DA 70+ → Google 이 상위 노출
  • DA 0~10 → 검색 결과 뒤
  • DA 는 시간 + 백링크 자연 축적 만이 답

저희 같은 신규 도메인은 DA 가 거의 0 에 가깝습니다. 이전 taystudio.github.io 의 DA 가 일부 승계됐을 것으로 추정은 하지만, 어쨌든 3-6개월의 축적이 필요한 부분입니다.

4. AI Overview — Google 검색의 새로운 적

2024년부터 Google 검색 결과 상단에 AI 가 직접 답을 보여주기 시작했습니다.

검색: "연봉 5000만 실수령액"
↓
AI Overview: "약 354만원 (4대보험·소득세 차감 후)..."
↓
사용자가 우리 사이트 클릭 안 함

저희 같은 계산기 사이트에는 직격타입니다. 그나마 Naver 는 아직 한국 정책 데이터에 대한 AI 학습이 약한 편이라 영향이 미미합니다.

5. Zero-Click Search

검색은 했지만 사이트를 클릭하지 않고, 검색 결과 페이지에서 답만 보고 나가는 경우를 말합니다.

  • Featured Snippet (강조 박스)
  • Knowledge Panel (지식 패널)
  • AI Overview
  • "현재 Google 검색의 60%+ 가 zero-click" (Similarweb 2024)

대응은 단순한 fact 답변 너머의 차별화 콘텐츠입니다. 시뮬레이션·매트릭스·case study·시각화처럼 AI 가 한 줄로 답할 수 없는 깊이를 만드는 것이죠.

저희가 도구에 매트릭스(1주택 공정시장가액 9×4 / DSR 역산 / 6+6 분배 시나리오)를 넣은 이유가 정확히 이것입니다.

6. 한국 검색 시장 구조

엔진 점유율 (한국) 특징
Naver 50~60% 한국 NLP 강력, 자체 데이터 우선
Google 30~35% 글로벌, PageRank 유산, AI Overview
Daum (Kakao) 5~10% Google 결과 부분 + 자체 카페
Bing 1~3% Microsoft. GSC import 가능

세금·계산기·정책 같은 한국어 키워드는 Naver 가 확실히 우위입니다. 그래서 Google 만 보고 사이트를 평가하면 안 된다는 게 제가 한 달 운영하며 가장 절실히 느낀 부분입니다.

7. Sitemap·robots.txt·llms.txt — 검색엔진과의 약속 문서

파일 역할
sitemap.xml "내 사이트의 모든 페이지 list 여기 있다"
robots.txt "이 부분 크롤 OK / 이 부분 X"
llms.txt "LLM 학습·검색용 사이트 메타" (신생 표준)
IndexNow 페이지 변경 시 검색엔진 즉시 알림 (Bing·Yandex·Naver)

요즘은 robots.txt 에서 GPTBot·ClaudeBot·PerplexityBot·Google-Extended 를 명시적으로 Allow 해 주는 것이 표준입니다. 저희도 그렇게 설정해 두었습니다.

8. Schema.org / JSON-LD — Rich Snippet 의 원리

검색엔진이 이해할 수 있도록 만든 구조화 데이터 표준입니다. 그중 JSON-LD 형식이 가장 흔하게 쓰입니다.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "연봉 5000만의 실수령액?",
    "acceptedAnswer": { "@type": "Answer", "text": "월 약 354만원..." }
  }]
}
</script>

이걸 넣어 두면 Google 검색 결과에 별점·FAQ 펼치기·빵부스러기 경로 같은 것이 표시되면서 CTR 이 올라갑니다.

9. CTR Optimization — 노출 → 클릭 비율

검색 결과 페이지에 우리 사이트가 떠도 클릭이 안 되면 의미가 없습니다. CTR 을 좌우하는 요인은 이렇습니다.

  • <title> 태그: 검색어 직접 매칭 + 매력적
  • <meta name="description">: 150자 안에 핵심 가치
  • URL 가독성: /tools/salary/ > /p?id=12345
  • rich snippet 표시: ⭐별점·FAQ 펼치기
  • 사이트 신뢰도 (HTTPS, 도메인 age)

저희는 신규 도메인에 AI Overview 영향까지 겹쳐서 GSC CTR 이 0.3% 였습니다(정상은 2-5%). 그런데 같은 사이트인데 Naver 에서는 0.9% 로, 3배 차이가 났습니다. 한국 검색엔진이 한국 콘텐츠를 더 잘 매칭해 준다는 걸 숫자로 확인한 셈입니다.

10. 키워드 — Short-tail vs Long-tail

유형 의미 예시
Short-tail 짧고 broad. 검색량 많지만 경쟁 치열 "재산세"
Long-tail 구체적·길고 검색량 작지만 의도 명확 "지방세법 주택 재산세 세율 2026"

신규 사이트는 short-tail 경쟁에서 이길 수가 없습니다. 결국 long-tail 을 꾸준히 누적하는 게 정도입니다. 저희가 매트릭스와 시뮬레이션을 계속 더하는 것도 long-tail 키워드를 흡수하려는 전략입니다.

11. 분석 도구 3종 — 한 쪽만 보면 큰 그림 못 봄

도구 보여주는 것 한계
GSC Google 검색 노출·클릭·색인 Google 만
Naver SearchAdvisor Naver 검색 동일 Naver 만
Cloudflare Web Analytics 모든 채널 실제 트래픽 검색어 정보 X

이 3종을 합쳐 보니 그림이 이렇게 나왔습니다.

  • GSC (3개월): 클릭 2 / 노출 777
  • Naver (30일): 클릭 230 / 노출 26,000
  • Cloudflare (24h): visits 57

Naver 가 GSC 의 33~115배였던 겁니다. Google 만 봤더라면 사이트가 망한 줄 알고 접었을지도 모릅니다.

12. GEO (Generative Engine Optimization) — AI 검색 시대 SEO

ChatGPT·Claude·Perplexity·Gemini 가 답할 때 우리 사이트를 인용하게 만드는 최적화입니다.

  • llms.txt — 사이트 root markdown 메타
  • 구조화 데이터 (Schema.org)
  • 출처 명시 (author·date·source)
  • fact-rich content (수치·출처·정확성)

저희 Cloudflare 에서 chatgpt.com referrer 가 이미 확인됐습니다. LLM 쪽 노출이 시작됐다는 뜻이고, llms.txt 를 갱신하면 더 가속될 것으로 기대하고 있습니다.

정리 — SEO 80% 는 시간, 20% 가 액션

운영자가 할 수 있는 것은 이 정도입니다. - sitemap·robots·llms.txt 설정 - 모든 검색엔진 등록 (GSC·Naver·Daum·Bing) - meta·title CTR 최적화 - 구조화 데이터 - 콘텐츠 차별화

반대로 할 수 없는 것은 이렇습니다. - Google sandbox 단축 - DA 즉시 상승 - AI Overview 완화 - Naver 알고리즘 변경

신규 도메인을 한 달 운영하고 내린 결론은, 데이터 한쪽만 보고 패닉하지 말자는 것입니다. Google 이 약하더라도 Naver·Daum·AI 채널을 합치면 충분한 트래픽이 들어옵니다. 저희 사이트만 해도 일 방문 57, 한 달 Naver 클릭 230, AI 검색 노출 시작까지 — 절대 망한 상태가 아니었습니다.
